## 왜 Claude Code인가?

> **"Google Antigravity (Jules), Cursor, Windsurf... 다른 도구도 많은데 왜 Claude Code?"**

Claude Code를 기반으로 커리큘럼을 구성한 이유:

| 기준 | Claude Code | 다른 도구들 |
|------|-------------|------------|
| **터미널 네이티브** | ✅ CLI 기반, 어디서든 동작 | IDE 종속적인 경우 많음 |
| **투명한 동작** | ✅ 모든 행동을 명시적으로 보여줌 | 블랙박스인 경우 있음 |
| **학습 가치** | ✅ 원리를 이해하며 배움 | 편의성에 원리가 가려짐 |
| **확장성** | ✅ Hooks, MCP, Skills 등 | 도구마다 다름 |
| **비용** | ✅ API 사용량 기반 | 구독 모델 다수 |

### 다른 도구와의 호환성

**좋은 소식**: 이 커리큘럼에서 배우는 핵심 개념들은 도구에 구애받지 않습니다!

- **프롬프팅 기법** → 모든 AI 도구에 적용
- **바이브코딩 사이클** → 도구 불문
- **프로젝트 구조 이해** → 개발의 기본
- **Git 워크플로우** → 표준

> 💡 **향후 계획**: Cursor, Windsurf, Jules 등 다른 도구 커리큘럼도 확장 예정입니다.

## 바이브코딩이란?

**바이브코딩**은 AI와 함께 개발하는 새로운 패러다임입니다. Andrej Karpathy(전 Tesla AI 책임자)가 처음 사용한 용어입니다.

```
전통적인 코딩:
당신 → 문법 학습 → 코드 작성 → 디버깅 → 소프트웨어

바이브코딩:
당신 → 의도 표현 → AI가 코드 작성 → 함께 개선 → 소프트웨어
```

### 바이브코딩의 3단계 사이클

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   1. 의도 전달  →  2. 결과 확인  →  3. 피드백으로 개선      │
│        ↑                                        │          │
│        └────────────────────────────────────────┘          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

1. **의도 전달**: Claude에게 원하는 것을 자연어로 설명
2. **결과 확인**: 기대한 대로 됐는지 확인
3. **피드백으로 개선**: 부족한 점을 알려주고 수정

완벽해질 때까지 반복. 문법 암기 필요 없음.

<details>
<summary><strong>바이브코딩 실제 모습 보기</strong></summary>

### 예시: 웹페이지 만들기

**전통적인 코딩 (수시간의 학습 필요):**
```html
<!DOCTYPE html>
<html>
<head>
    <title>내 페이지</title>
    <style>
        body { font-family: sans-serif; }
        h1 { color: blue; }
    </style>
</head>
<body>
    <h1>안녕하세요</h1>
</body>
</html>
```

**바이브코딩 (30초):**
```
나: 파란색 제목으로 '안녕하세요'가 나오는 웹페이지 만들어줘
Claude: [파일 즉시 생성]
나: 제목 더 크게 하고 가운데 정렬해줘
Claude: [파일 수정]
```

AI가 문법을 처리합니다. 당신은 **무엇**을 원하는지에만 집중하세요. **어떻게** 코딩하는지가 아니라.

</details>

## Pro Tips

Claude Code 팀과 숙련된 개발자들의 팁입니다:

### 필수 팁

1. **Plan 모드로 시작** - `Shift+Tab` 두 번. Claude가 먼저 생각하게.
2. **검증 방법 명시** - "변경 후 테스트 실행해줘"
3. **AI 코드도 리뷰** - 사람 코드와 같은 기준 적용
4. **CLAUDE.md를 Git에 커밋** - Claude가 실수하면 업데이트
5. **여러 터미널 사용** - 복잡한 작업은 병렬로

### Claude Code vs 웹 채팅

| 기능 | 웹 채팅 | Claude Code |
|------|---------|-------------|
| 파일 접근 | ❌ 복사-붙여넣기 | ✅ 직접 읽기/쓰기 |
| 명령어 실행 | ❌ 수동 | ✅ 자동 |
| 프로젝트 맥락 | ❌ 매번 설명 | ✅ 구조 파악 |
| 연속 작업 | ❌ 세션 기반 | ✅ 지속적 |

**요약**: 웹 채팅은 컨설턴트. Claude Code는 터미널 안의 페어 프로그래머.
