# 용어 사전 (Glossary)

> 이 문서는 커리큘럼에서 자주 등장하는 용어들을 비전공자도 이해하기 쉽게 설명합니다.
> 모르는 용어가 나오면 여기서 찾아보세요!

## 🖥️ 컴퓨터 기초

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **터미널 (Terminal)** | 컴퓨터에 글자로 명령하는 검은색/흰색 창 | 카카오톡처럼 문자로 대화하는 메신저, 단 상대방이 컴퓨터 |
| **CLI** | Command Line Interface. 터미널과 같은 의미 | 글자로 명령하는 방식 |
| **GUI** | Graphical User Interface. 마우스로 클릭하는 방식 | 아이콘 클릭하는 일반적인 방식 |
| **폴더/디렉토리 (Folder/Directory)** | 파일을 담는 공간 | 서랍장의 칸 |
| **경로 (Path)** | 파일이나 폴더의 위치를 나타내는 주소 | 집 주소 (예: 서울시 강남구 역삼동) |
| **확장자 (Extension)** | 파일 이름 뒤의 .txt, .html 같은 부분 | 파일의 종류를 알려주는 꼬리표 |
| **루트 (Root)** | 최상위 폴더. Mac/Linux는 `/`, Windows는 `C:\` | 건물의 1층 |
| **홈 디렉토리** | 사용자 개인 폴더 | 내 방 |
| **환경 변수 (Environment Variable)** | 시스템 전체에서 사용하는 설정값 | 컴퓨터가 항상 기억하는 메모 |
| **실행 파일 (Executable)** | 프로그램으로 실행할 수 있는 파일 | 켤 수 있는 기계 |
| **프로세스 (Process)** | 실행 중인 프로그램 | 현재 돌아가고 있는 기계 |
| **권한 (Permissions)** | 파일을 읽고, 쓰고, 실행할 수 있는 규칙 | 문의 열쇠와 자물쇠 |
| **셸 (Shell)** | 터미널 명령어를 해석하는 프로그램 | 당신과 컴퓨터 사이의 통역사 |
| **Bash/Zsh** | 인기 있는 셸 프로그램들 | 다른 통역사들 (약간의 억양 차이) |
| **stdin/stdout/stderr** | 표준 입력, 출력, 오류 스트림 | 키보드, 화면, 오류 메시지 채널 |
| **파이프 (\|)** | 한 명령어의 출력을 다른 명령어로 보냄 | 기계들 사이의 컨베이어 벨트 |
| **플래그/옵션 (Flag/Option)** | 명령어 동작을 바꾸는 수정자 (예: `-v`, `--help`) | 기계의 설정 다이얼 |

## 🌐 웹 기초

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **HTML** | HyperText Markup Language. 웹페이지의 구조와 내용을 담당 | 집의 뼈대와 구조 |
| **CSS** | Cascading Style Sheets. 웹페이지의 디자인과 스타일을 담당 | 집의 인테리어, 페인트, 가구 배치 |
| **JavaScript (JS)** | 웹페이지의 동작을 담당하는 프로그래밍 언어 | 집의 전기, 수도, 엘리베이터 |
| **브라우저 (Browser)** | 웹페이지를 보는 프로그램 (Chrome, Safari 등) | TV (웹사이트가 방송이라면) |
| **서버 (Server)** | 웹사이트를 저장하고 보내주는 컴퓨터 | 식당 주방 |
| **클라이언트 (Client)** | 콘텐츠를 요청하고 표시하는 사용자의 브라우저/기기 | 식당 손님 |
| **배포 (Deploy)** | 만든 웹사이트를 인터넷에 올리는 것 | 방송 송출 시작 |
| **로컬 (Local)** | 내 컴퓨터 | 집 |
| **리모트 (Remote)** | 다른 컴퓨터/서버 | 다른 곳 |
| **호스팅 (Hosting)** | 웹사이트를 인터넷에 올려두는 서비스 | 가게 임대 서비스 |
| **도메인 (Domain)** | 웹사이트 주소 (예: google.com) | 가게 간판 |
| **URL** | 전체 웹 주소 (https://www.example.com/page) | 완전한 주소 |
| **HTTPS** | HTTP의 보안 버전 (자물쇠 아이콘) | 봉인된 봉투로 편지 보내기 |
| **반응형 (Responsive)** | 화면 크기에 따라 레이아웃이 바뀌는 것 | 어떤 체형에도 맞는 옷 |
| **뷰포트 (Viewport)** | 브라우저에서 보이는 웹페이지 영역 | 창문으로 보이는 부분 |
| **DOM** | Document Object Model. 브라우저의 HTML 표현 | 페이지 요소들의 가계도 |
| **렌더링 (Rendering)** | 코드를 화면에 보이는 것으로 변환 | 재료로 요리 만들기 |
| **캐시 (Cache)** | 빠른 접근을 위해 저장된 데이터 사본 | 냉장고에 남은 음식 보관 |
| **쿠키 (Cookie)** | 웹사이트가 브라우저에 저장하는 작은 데이터 | 웹사이트가 기억하는 이름표 |
| **세션 (Session)** | 웹사이트에서의 사용자 활동 기간 | 가게 한 번 방문 |
| **CDN** | Content Delivery Network. 빠른 콘텐츠 전달을 위한 전세계 서버 | 빠른 배송을 위한 여러 창고 위치 |

## 📦 개발 도구

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **npm** | Node Package Manager. JavaScript 도구/라이브러리 설치 프로그램 | 코드용 앱스토어 |
| **yarn/pnpm** | npm의 대체 패키지 매니저 | 다른 앱스토어, 같은 앱들 |
| **Node.js** | 브라우저 밖에서 JavaScript를 실행하게 해주는 프로그램 | 컴퓨터용 JavaScript 통역사 |
| **패키지 (Package)** | 다른 사람이 만든 재사용 가능한 코드 묶음 | 레고 조립 세트 |
| **라이브러리 (Library)** | 자주 쓰는 기능을 모아둔 코드 | 요리 양념 세트 |
| **프레임워크 (Framework)** | 개발의 틀을 잡아주는 큰 라이브러리 | 프랜차이즈 매뉴얼 |
| **의존성 (Dependency)** | 프로젝트가 필요로 하는 코드 | 요리에 필요한 재료 |
| **package.json** | 프로젝트의 의존성과 설정을 나열한 파일 | 쇼핑 목록 + 레시피 카드 |
| **node_modules** | 설치된 모든 패키지가 들어있는 폴더 | 재료가 가득 찬 식료품 저장실 |
| **Tailwind CSS** | 유틸리티 우선 CSS 프레임워크 | 빠른 바느질을 위한 미리 잘린 천 조각 |
| **React** | UI 구축을 위한 인기 JavaScript 라이브러리 | 웹 인터페이스용 레고 블록 |
| **Next.js** | 추가 기능이 있는 React 프레임워크 | React + 보너스 기능 번들 |
| **Vue/Angular/Svelte** | 다른 인기 프론트엔드 프레임워크들 | 다른 브랜드의 레고 |
| **VS Code** | 인기 있는 코드 편집 프로그램 | 전문가용 메모장 |
| **IDE** | Integrated Development Environment. 개발에 필요한 도구를 모아둔 프로그램 | 완전한 공구 세트 |
| **린터 (Linter)** | 코드의 오류와 스타일을 검사하는 도구 | 코드용 맞춤법 검사기 |
| **Prettier** | 코드를 자동으로 정리하는 도구 | 코드 스타일링 자동 수정 |
| **TypeScript** | 타입 검사가 있는 JavaScript | 보조 바퀴 달린 JavaScript |
| **트랜스파일 (Transpile)** | 한 언어 버전에서 다른 버전으로 코드 변환 | 현대 속어를 고전 언어로 번역 |
| **번들 (Bundle)** | 여러 파일을 하나로 합치기 | 여행 가방 싸기 |
| **미니파이 (Minify)** | 코드에서 불필요한 문자 제거 | 파일 압축 |
| **Webpack/Vite/esbuild** | 코드를 번들링하고 최적화하는 빌드 도구 | 재료를 준비하는 주방 기기 |

## 🔌 API와 통신

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **API** | Application Programming Interface. 프로그램끼리 대화하는 방법 | 식당 주문 시스템 (메뉴판 + 주문서) |
| **요청 (Request)** | 서버에 무언가를 달라고 보내는 것 | 주문하기 |
| **응답 (Response)** | 서버가 보내주는 결과 | 음식 받기 |
| **엔드포인트 (Endpoint)** | API에 접근할 수 있는 특정 URL | 서비스 센터의 특정 창구 |
| **JSON** | JavaScript Object Notation. 데이터를 주고받는 형식 | 모두가 읽을 수 있는 표준 양식 |
| **REST** | 일반적인 API 디자인 스타일 | 메뉴가 구성되는 표준 방식 |
| **GraphQL** | 대안적인 API 쿼리 언어 | "직접 만드는" 메뉴 |
| **HTTP 메서드** | GET(읽기), POST(생성), PUT(수정), DELETE(삭제) | 창구에서의 다른 요청 유형들 |
| **상태 코드 (Status Code)** | 요청 결과를 나타내는 숫자 (200=OK, 404=Not Found, 500=Error) | 성공/실패 정보가 담긴 영수증 |
| **헤더 (Headers)** | 요청/응답과 함께 보내는 메타데이터 | 편지를 감싸는 봉투 |
| **바디 (Body)** | 요청/응답의 주요 내용 | 봉투 안의 편지 |
| **인증 (Authentication)** | 누구인지 증명하기 | 신분증 보여주기 |
| **권한 부여 (Authorization)** | 무엇을 할 수 있는지 확인 | 멤버십 등급 확인 |
| **토큰 (Token)** | 인증되었음을 증명하는 문자열 | 축제에서의 팔찌 |
| **API 키** | API에 접근하기 위한 비밀번호 | 특정 문의 열쇠 |
| **레이트 리미팅 (Rate Limiting)** | 요청 횟수 제한 | "1인 1개" 규칙 |
| **웹훅 (Webhook)** | 무언가 발생했을 때 자동 알림 | 콜백 전화번호 |
| **웹소켓 (WebSocket)** | 양방향 실시간 통신 | 전화 통화 (문자 vs) |
| **CORS** | Cross-Origin Resource Sharing. 웹 요청 보안 | 웹사이트 간 국경 통제 |

## 🔧 Git과 버전 관리

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **Git** | 코드의 변경 이력을 관리하는 도구 | 무한 되돌리기 + 타임머신 |
| **GitHub** | Git 저장소를 온라인에 보관하는 서비스 | 코드용 클라우드 드라이브 |
| **GitLab/Bitbucket** | GitHub의 대안들 | 다른 클라우드 저장소 제공업체 |
| **저장소 (Repository)** | Git으로 추적되는 프로젝트 폴더 | 전체 히스토리가 있는 프로젝트 폴더 |
| **커밋 (Commit)** | 변경사항의 스냅샷을 저장하는 것 | 게임 세이브 포인트 |
| **푸시 (Push)** | 커밋을 GitHub에 업로드 | 클라우드에 동기화 |
| **풀 (Pull)** | GitHub에서 변경사항 다운로드 | 클라우드에서 다운로드 |
| **클론 (Clone)** | 저장소를 내 컴퓨터로 복사 | 프로젝트 다운로드 |
| **포크 (Fork)** | 다른 사람 저장소의 내 복사본 만들기 | 나만의 리믹스 만들기 |
| **브랜치 (Branch)** | 코드의 평행 버전 | 실험용 세이브 슬롯 |
| **메인/마스터 (Main/Master)** | 주요 브랜치 | 공식 버전 |
| **머지 (Merge)** | 브랜치들을 합치는 것 | 두 세이브 파일 합치기 |
| **머지 충돌 (Merge Conflict)** | Git이 변경사항을 자동 합치지 못할 때 | 두 사람이 같은 문장을 다르게 수정 |
| **PR (Pull Request)** | 변경사항을 머지해달라는 요청 | "내 버전으로 할까요?" |
| **코드 리뷰** | 머지 전에 다른 사람이 코드 검토 | 에세이 동료 검토 |
| **체크아웃 (Checkout)** | 다른 브랜치로 전환 | 다른 세이브 파일 로드 |
| **스태시 (Stash)** | 커밋하지 않은 변경사항 임시 저장 | 서랍에 임시 보관 |
| **리베이스 (Rebase)** | 커밋 히스토리 다시 쓰기 | 이야기의 타임라인 재구성 |
| **체리픽 (Cherry-pick)** | 다른 브랜치에서 특정 커밋만 적용 | 다른 요리책에서 레시피 하나만 복사 |
| **HEAD** | 현재 커밋을 가리키는 포인터 | "현재 위치" 표시 |
| **오리진 (Origin)** | 원격 저장소의 기본 이름 | 홈 베이스 |
| **.gitignore** | Git이 무시해야 할 파일 목록 | "싸지 마세요" 목록 |

## 🤖 AI와 Claude Code

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **LLM** | Large Language Model. 대규모 언어 모델 | 수백만 권의 책을 읽은 AI |
| **프롬프트 (Prompt)** | AI에게 하는 요청/질문 | AI에게 하는 말 |
| **컨텍스트 (Context)** | AI가 대화 중 기억하고 있는 정보 | AI의 단기 기억 |
| **컨텍스트 윈도우** | AI가 한 번에 고려할 수 있는 최대 텍스트 양 | AI 작업 메모리의 크기 |
| **토큰 (Token)** | 텍스트 단위 (대략 4자 또는 단어의 3/4) | 텍스트의 음절 |
| **모델 (Model)** | AI의 "두뇌" 버전 (Opus, Sonnet, Haiku) | 다른 스킬 레벨/속도 |
| **추론 (Inference)** | AI가 응답을 생성하는 것 | AI가 생각하고 답변하기 |
| **파인튜닝 (Fine-tuning)** | 특정 데이터로 AI 훈련 | 특정 과목 전문화 |
| **에이전트 (Agent)** | 스스로 판단하고 행동하는 AI | 자율 주행 모드 |
| **서브에이전트 (Subagent)** | 큰 작업을 나눠서 처리하는 작은 AI | 하위 작업 담당 팀원 |
| **도구 사용 (Tool Use)** | AI가 외부 도구 사용 (검색, 코드 실행) | AI가 앱 사용 |
| **Plan 모드** | 읽기만 하고 수정하지 않는 모드 | 보기만 하고 만지지 않기 |
| **자동 승인 모드 (Auto-accept)** | 파일 수정을 자동 승인하는 모드 | 신뢰 모드 |
| **CLAUDE.md** | 프로젝트 규칙을 적어두는 파일 | AI를 위한 업무 지침서 |
| **Hooks** | 특정 상황에 자동 실행되는 스크립트 | 자동 트리거 |
| **MCP** | Model Context Protocol. 외부 도구 연결 | AI용 USB 포트 |
| **Skills** | 재사용 가능한 Claude Code 기능 | 저장된 단축키 |
| **슬래시 커맨드** | /로 시작하는 명령어 (/help 같은) | 키보드 단축키 |
| **에이전틱 코딩** | AI가 자율적으로 코드 작성 및 실행 | AI 페어 프로그래머 |
| **환각 (Hallucination)** | AI가 틀린 정보를 자신있게 생성할 때 | AI의 상상력이 너무 나갈 때 |
| **온도 (Temperature)** | AI 응답의 무작위성 조절 | 창의성 다이얼 |
| **시스템 프롬프트** | AI 동작을 안내하는 숨겨진 지시사항 | 뒤에서 작동하는 규칙 |

## 💻 프로그래밍 기초

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **변수 (Variable)** | 값을 저장하는 이름 붙은 컨테이너 | 라벨이 붙은 상자 |
| **상수 (Constant)** | 바꿀 수 없는 변수 | 봉인된 라벨 상자 |
| **함수 (Function)** | 특정 작업을 수행하는 재사용 가능한 코드 블록 | 레시피 |
| **매개변수/인자 (Parameter/Argument)** | 함수에 전달되는 값 | 레시피 재료 |
| **반환값 (Return Value)** | 함수가 돌려주는 것 | 완성된 요리 |
| **클래스 (Class)** | 객체를 만들기 위한 청사진 | 쿠키 틀 |
| **객체 (Object)** | 관련된 데이터와 함수의 모음 | 실제 쿠키 (속성 포함) |
| **메서드 (Method)** | 객체에 속한 함수 | 객체가 할 수 있는 행동 |
| **프로퍼티 (Property)** | 객체에 속한 값 | 객체의 특성 |
| **배열 (Array)** | 순서가 있는 값들의 목록 | 번호가 매겨진 목록 |
| **객체/딕셔너리** | 키-값 쌍의 모음 | 실제 사전 (단어 → 정의) |
| **루프 (Loop)** | 반복되는 코드 | 반복 재생되는 플레이리스트 |
| **조건문 (Conditional)** | 무언가 참일 때만 실행되는 코드 | "만약-그러면" 규칙 |
| **불리언 (Boolean)** | 참 또는 거짓 값 | 예 또는 아니오 |
| **문자열 (String)** | 텍스트 데이터 | 따옴표 안의 글자들 |
| **정수 (Integer)** | 소수점 없는 숫자 | 세는 숫자 |
| **실수 (Float)** | 소수점 있는 숫자 | 소수점이 있는 숫자 |
| **Null/Undefined** | 없음 또는 설정되지 않음 | 비어있거나 존재하지 않음 |
| **버그 (Bug)** | 코드의 오류 | 오타, 실수 |
| **디버깅 (Debugging)** | 버그를 찾아서 고치는 것 | 교정하고 수정하기 |
| **컴파일 (Compile)** | 코드를 컴퓨터가 읽을 수 있는 형식으로 변환 | 책 번역 |
| **런타임 (Runtime)** | 프로그램이 실행되는 중 | 공연 중 |
| **에러/예외 (Error/Exception)** | 실행 중 문제 발생 | 기계 고장 |
| **스택 트레이스 (Stack Trace)** | 오류로 이어지는 함수 호출 경로 | 버그로 가는 빵 부스러기 흔적 |
| **로그 (Log)** | 프로그램 활동 기록 | 일기장 |
| **콘솔 (Console)** | 로그와 오류가 나타나는 곳 | 게시판 |
| **문법 (Syntax)** | 프로그래밍 언어의 문법 규칙 | 맞춤법과 문법 |
| **시맨틱 (Semantic)** | 코드의 의미 | 문장이 의미하는 것 |
| **알고리즘 (Algorithm)** | 단계별 문제 해결 지침 | 자세한 레시피 |
| **자료 구조 (Data Structure)** | 데이터를 구성하는 방법 | 파일링 시스템 |
| **재귀 (Recursion)** | 자기 자신을 호출하는 함수 | 러시안 마트료시카 인형 |
| **콜백 (Callback)** | 나중에 실행하기 위해 다른 함수에 전달하는 함수 | "끝나면 전화해" |
| **프로미스 (Promise)** | 미래의 값을 나타냄 | 차용증 |
| **Async/Await** | 시간이 걸리는 작업을 처리하는 방법 | "그거 끝날 때까지 기다릴게" |
| **스코프 (Scope)** | 변수에 접근할 수 있는 곳 | 가시 범위 |
| **전역 (Global)** | 어디서나 접근 가능 | 공개 |
| **지역 (Local)** | 특정 영역에서만 접근 가능 | 비공개 |

## 🗄️ 데이터베이스

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **데이터베이스 (DB)** | 데이터를 체계적으로 저장하는 곳 | 거대한 스프레드시트 시스템 |
| **SQL** | Structured Query Language. 데이터베이스 언어 | 데이터베이스가 사용하는 언어 |
| **NoSQL** | 비관계형 데이터베이스 | 엄격한 테이블 없는 데이터베이스 |
| **테이블 (Table)** | 행과 열로 된 관련 데이터 모음 | 엑셀 시트 |
| **행/레코드 (Row/Record)** | 테이블의 단일 항목 | 스프레드시트의 한 줄 |
| **열/필드 (Column/Field)** | 데이터 범주 | 열 제목 |
| **기본 키 (Primary Key)** | 각 행의 고유 식별자 | 학번 |
| **외래 키 (Foreign Key)** | 다른 테이블의 기본 키 참조 | "관련 파일 #123 참조" |
| **쿼리 (Query)** | 데이터베이스에서 데이터 요청 | 검색 질문 |
| **CRUD** | Create, Read, Update, Delete - 기본 데이터베이스 작업 | 추가, 보기, 수정, 삭제 |
| **스키마 (Schema)** | 데이터베이스의 구조/설계 | 스프레드시트 템플릿 |
| **마이그레이션 (Migration)** | 데이터베이스 구조를 통제된 방식으로 변경 | 내용물 유지하면서 리모델링 |
| **인덱스 (Index)** | 데이터 검색 속도 향상 | 책의 색인 |
| **ORM** | Object-Relational Mapping. 데이터베이스와 대화하는 코드 | 코드와 데이터베이스 사이 통역사 |
| **PostgreSQL/MySQL/SQLite** | 인기 있는 SQL 데이터베이스 | 다른 브랜드의 파일 캐비닛 |
| **MongoDB** | 인기 있는 NoSQL 데이터베이스 | 더 유연한 파일링 시스템 |
| **Supabase** | 추가 기능이 있는 데이터베이스 서비스 | 슈퍼파워가 있는 온라인 스프레드시트 |
| **Redis** | 빠른 인메모리 데이터베이스 | 빠른 접근을 위한 포스트잇 |

| **VPN** | Virtual Private Network. 암호화된 터널 | 비밀 통로 |

## 📚 빠른 참조 카드

### 터미널 명령어 치트시트

| 명령어 | 하는 일 |
|--------|--------|
| `cd 폴더` | 디렉토리 이동 |
| `ls` / `dir` | 파일 목록 |
| `pwd` | 현재 디렉토리 출력 |
| `mkdir 이름` | 새 폴더 만들기 |
| `rm 파일` | 파일 삭제 |
| `cp a b` | a를 b로 복사 |
| `mv a b` | a를 b로 이동/이름변경 |
| `cat 파일` | 파일 내용 보기 |

### Git 명령어 치트시트

| 명령어 | 하는 일 |
|--------|--------|
| `git init` | 새 저장소 시작 |
| `git clone url` | 저장소 복사 |
| `git status` | 변경사항 보기 |
| `git add .` | 모든 변경사항 스테이징 |
| `git commit -m "메시지"` | 스냅샷 저장 |
| `git push` | GitHub에 업로드 |
| `git pull` | GitHub에서 다운로드 |
| `git branch 이름` | 브랜치 생성 |
| `git checkout 이름` | 브랜치 전환 |

### npm 명령어 치트시트

| 명령어 | 하는 일 |
|--------|--------|
| `npm init` | 새 프로젝트 시작 |
| `npm install` | 모든 의존성 설치 |
| `npm install 패키지` | 패키지 설치 |
| `npm run 스크립트` | 스크립트 실행 |
| `npm start` | start 스크립트 실행 |
| `npm test` | 테스트 실행 |
| `npx 명령어` | 설치 없이 실행 |
