# 용어 사전 (Glossary)

> 이 문서는 커리큘럼에서 자주 등장하는 용어들을 비전공자도 이해하기 쉽게 설명합니다.
> 모르는 용어가 나오면 여기서 찾아보세요!

---

## 🖥️ 컴퓨터 기초

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **터미널 (Terminal)** | 컴퓨터에 글자로 명령하는 검은색/흰색 창 | 카카오톡처럼 문자로 대화하는 메신저, 단 상대방이 컴퓨터 |
| **CLI** | Command Line Interface. 터미널과 같은 의미 | 글자로 명령하는 방식 |
| **GUI** | Graphical User Interface. 마우스로 클릭하는 방식 | 아이콘 클릭하는 일반적인 방식 |
| **폴더 (Folder)** | 파일을 담는 공간. 디렉토리라고도 함 | 서랍장의 칸 |
| **경로 (Path)** | 파일이나 폴더의 위치를 나타내는 주소 | 집 주소 (예: 서울시 강남구 역삼동) |
| **확장자** | 파일 이름 뒤의 .txt, .html 같은 부분 | 파일의 종류를 알려주는 꼬리표 |
| **루트 (Root)** | 최상위 폴더. Mac은 `/`, Windows는 `C:\` | 건물의 1층 |
| **홈 디렉토리** | 사용자 개인 폴더 | 내 방 |

## 🌐 웹 관련

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **HTML** | 웹페이지의 구조와 내용을 담당 | 집의 뼈대와 구조 |
| **CSS** | 웹페이지의 디자인과 스타일을 담당 | 집의 인테리어, 페인트, 가구 배치 |
| **JavaScript (JS)** | 웹페이지의 동작을 담당 | 집의 전기, 수도, 엘리베이터 |
| **브라우저** | 웹페이지를 보는 프로그램 (Chrome, Safari 등) | TV (웹사이트가 방송이라면) |
| **서버** | 웹사이트를 저장하고 보내주는 컴퓨터 | 방송국 |
| **배포 (Deploy)** | 만든 웹사이트를 인터넷에 올리는 것 | 방송 송출 시작 |
| **로컬 (Local)** | 내 컴퓨터 | 집 |
| **호스팅** | 웹사이트를 인터넷에 올려두는 서비스 | 부동산 임대 |
| **도메인** | 웹사이트 주소 (예: google.com) | 가게 간판 |
| **URL** | 전체 웹 주소 (https://...) | 가게의 전체 주소 |
| **반응형 (Responsive)** | 화면 크기에 따라 레이아웃이 바뀌는 것 | 접이식 테이블 |

## 📦 개발 도구

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **npm** | JavaScript 도구/라이브러리 설치 프로그램 | 앱스토어 |
| **Node.js** | 컴퓨터에서 JavaScript를 실행하게 해주는 프로그램 | JavaScript 통역사 |
| **패키지 (Package)** | 다른 사람이 만든 코드 묶음 | 레고 조립 세트 |
| **라이브러리** | 자주 쓰는 기능을 모아둔 코드 | 요리 양념 세트 |
| **프레임워크** | 개발의 틀을 잡아주는 큰 라이브러리 | 프랜차이즈 매뉴얼 |
| **Tailwind CSS** | CSS를 쉽게 쓰게 해주는 도구 | 인테리어 스티커 |
| **VS Code** | 코드 편집 프로그램 | 전문가용 메모장 |
| **IDE** | 개발에 필요한 도구를 모아둔 프로그램 | 공구 세트 |

## 🔌 API와 통신

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **API** | 프로그램끼리 대화하는 방법 | 식당 주문 시스템 (메뉴판 + 주문서) |
| **요청 (Request)** | 서버에 무언가를 달라고 보내는 것 | 주문하기 |
| **응답 (Response)** | 서버가 보내주는 결과 | 음식 받기 |
| **JSON** | 데이터를 주고받는 형식 | 주문서 양식 |
| **토큰 (Token)** | 사용량을 세는 단위 (AI 서비스에서) | 놀이공원 이용권 |
| **엔드포인트** | API의 특정 주소 | 특정 창구 |

## 🔧 Git과 버전 관리

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **Git** | 코드의 변경 이력을 관리하는 도구 | 무한 되돌리기가 가능한 저장 시스템 |
| **GitHub** | Git 저장소를 온라인에 보관하는 서비스 | 클라우드 드라이브 |
| **저장소 (Repository)** | 프로젝트 코드를 담는 공간 | 프로젝트 폴더 (+ 히스토리) |
| **커밋 (Commit)** | 변경사항을 저장하는 것 | 게임 세이브 |
| **푸시 (Push)** | 내 컴퓨터의 커밋을 GitHub에 올리는 것 | 클라우드에 업로드 |
| **풀 (Pull)** | GitHub의 변경사항을 내 컴퓨터로 가져오는 것 | 클라우드에서 다운로드 |
| **브랜치 (Branch)** | 코드의 분기점, 평행 세계 | 게임의 다른 세이브 슬롯 |
| **머지 (Merge)** | 브랜치를 합치는 것 | 두 버전을 하나로 합치기 |
| **PR (Pull Request)** | 코드 변경을 검토 요청하는 것 | "이렇게 바꿔도 될까요?" |
| **클론 (Clone)** | 저장소를 복사해오는 것 | 복사 + 붙여넣기 |

## 🤖 AI와 Claude Code

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **LLM** | Large Language Model. 대규모 언어 모델 | 수많은 책을 읽은 AI |
| **프롬프트 (Prompt)** | AI에게 하는 요청/질문 | AI에게 하는 말 |
| **컨텍스트 (Context)** | AI가 기억하고 있는 대화 내용 | AI의 단기 기억 |
| **모델** | AI의 "두뇌" 버전 (Opus, Sonnet, Haiku) | 선수 등급 |
| **에이전트 (Agent)** | 스스로 판단하고 행동하는 AI | 자율 주행 모드 |
| **서브에이전트** | 큰 작업을 나눠서 처리하는 작은 AI | 팀원 |
| **Plan 모드** | 읽기만 하고 수정하지 않는 모드 | 견학 모드 |
| **Accept Edits 모드** | 파일 수정을 자동 승인하는 모드 | 신뢰 모드 |
| **CLAUDE.md** | 프로젝트 규칙을 적어두는 파일 | AI를 위한 업무 지침서 |
| **Hooks** | 특정 상황에 자동 실행되는 스크립트 | 자동 알림 설정 |
| **MCP** | Model Context Protocol. 외부 도구 연결 | AI용 USB 포트 |

## 💻 프로그래밍 일반

| 용어 | 설명 | 쉬운 비유 |
|------|------|----------|
| **변수 (Variable)** | 값을 저장하는 상자 | 이름표가 붙은 상자 |
| **함수 (Function)** | 특정 작업을 수행하는 코드 묶음 | 레시피 |
| **버그 (Bug)** | 코드의 오류 | 오타, 실수 |
| **디버깅 (Debugging)** | 버그를 찾아서 고치는 것 | 오류 수정 |
| **컴파일 (Compile)** | 코드를 컴퓨터가 이해하는 언어로 변환 | 번역 |
| **런타임 (Runtime)** | 프로그램이 실행되는 중 | 영화 상영 중 |
| **에러 (Error)** | 프로그램 실행 중 문제 발생 | 기계 고장 |
| **로그 (Log)** | 프로그램 실행 기록 | 일기장 |
| **타입 (Type)** | 데이터의 종류 (숫자, 글자 등) | 식재료 종류 |
| **스크립트** | 자동화된 명령어 모음 | 매크로 |
