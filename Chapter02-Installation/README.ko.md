## 왜 필요합니까?

Claude Code와 대화하려면 먼저 컴퓨터에 설치해야 합니다. 친구와 채팅하려면 메신저 앱을 먼저 다운로드하는 것과 같습니다.

**실제 상황**: 개인 웹사이트 아이디어가 있습니다. Claude가 만들어주길 원합니다. 하지만 먼저 Claude가 컴퓨터에 있어야 실제로 파일을 만들고 저장할 수 있습니다.

### 쉬운 비유: 비서 채용하기

코드를 잘 쓰는 뛰어난 비서를 고용했는데, 아직 사무실 밖에서 기다리고 있다고 상상해보십시오. 도움을 받으려면:

1. **문 열기** (터미널 열기)
2. **안으로 들어오게 하기** (Claude Code 설치)
3. **서로 소개하기** (로그인)

이번 챕터에서 하는 일이 바로 이겁니다.

### 왜 웹이 아니라 터미널인가요?

"그냥 claude.ai 웹사이트에서 대화하면 안 되나요?"라고 생각할 수 있습니다.

| 웹 Claude (claude.ai) | Claude Code (터미널) |
|----------------------|---------------------|
| 대화만 가능 | 대화 + 파일 생성/수정 |
| 코드를 복사해서 붙여넣기 | 코드를 직접 실행 |
| 프로젝트 구조 모름 | 전체 프로젝트 이해 |
| 한 번에 한 파일 | 여러 파일 동시 작업 |

Claude Code는 여러분의 컴퓨터에서 **직접 작업**합니다. 그래서 터미널이 필요합니다.

## 터미널이란?

터미널은 컴퓨터와 텍스트로 소통하는 창입니다.

평소에는 마우스로 폴더를 클릭하고, 앱 아이콘을 눌러서 실행합니다. 터미널은 같은 작업을 텍스트로 수행합니다.

```
# 예시
cd Documents        # Documents 폴더로 이동
ls                  # 현재 폴더의 파일 목록 보기
```

### GUI vs CLI: 두 가지 컴퓨터 사용법

컴퓨터와 소통하는 방법은 크게 두 가지입니다:

#### GUI (Graphical User Interface) - 그래픽 사용자 인터페이스

우리가 평소에 사용하는 방식입니다.
- 마우스로 폴더 아이콘 클릭
- 버튼 눌러서 앱 실행
- 드래그 앤 드롭으로 파일 이동

**장점:** 직관적, 배우기 쉬움
**단점:** 반복 작업이 번거로움, 자동화 어려움

#### CLI (Command Line Interface) - 명령줄 인터페이스

텍스트로 명령을 내리는 방식입니다.
- 키보드로 명령어 입력
- 결과가 텍스트로 출력
- 스크립트로 자동화 가능

**장점:** 빠름, 자동화 가능, 정밀한 제어
**단점:** 명령어를 알아야 함

> 💡 **비전공자 팁**
>
> 어려워 보이지만 걱정하지 마세요!
> Claude는 터미널 명령어도 수많이 학습했기 때문에, "이 폴더의 파일 목록 보여줘"라고 말하면 알아서 실행합니다.
> **터미널 여는 방법만 알면 됩니다.**

### 왜 개발자들은 터미널을 좋아할까요?

1. **속도**: 마우스를 움직이는 것보다 타이핑이 빠릅니다
2. **자동화**: 반복 작업을 스크립트로 만들 수 있습니다
3. **원격 작업**: 서버에 접속해서 작업할 때 터미널이 필수입니다
4. **정밀성**: 정확히 원하는 것만 할 수 있습니다

> 🔥 **프로 팁**
>
> 처음에는 터미널이 무섭게 느껴질 수 있지만, Claude Code를 사용하면 자연어로 말하면 되기 때문에 명령어를 외울 필요가 없습니다. 점점 익숙해지면 터미널의 강력함을 느끼게 될 것입니다.

### 터미널 여는 법

**Mac:**
1. `Command + Space`로 Spotlight 열기
2. "터미널" 또는 "Terminal" 입력
3. Enter

> 💡 **비전공자 팁 (Mac)**
>
> Spotlight가 익숙하지 않다면:
> 1. Finder를 엽니다
> 2. 상단 메뉴에서 "이동" > "유틸리티" 클릭
> 3. "터미널" 앱을 더블클릭
>
> 자주 사용하니 Dock에 고정해두면 편리합니다!

**Windows:**
1. `Windows 키` 누르기
2. "PowerShell" 입력
3. Enter

> 💡 **비전공자 팁 (Windows)**
>
> PowerShell과 명령 프롬프트(cmd) 두 가지가 있습니다. PowerShell이 더 현대적이니 PowerShell을 사용하세요.
> "관리자 권한으로 실행"이 아닌 일반 실행으로 시작하세요.

**Linux:**
1. `Ctrl + Alt + T` (대부분의 배포판)
2. 또는 앱 목록에서 "Terminal" 검색

### 추천 터미널 앱

기본 터미널도 충분하지만, 더 편리한 터미널 앱을 사용하면 작업이 훨씬 즐거워집니다.

#### Mac: iTerm2 (강력 추천)

기본 터미널 대신 [iTerm2](https://iterm2.com)를 추천합니다.

```
┌─────────────────────────────────────────────────────────────────┐
│                    iTerm2 vs 기본 터미널                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ✅ 화면 분할: 한 창에서 여러 터미널을 나눠 볼 수 있어요         │
│     (Claude Code 실행하면서 다른 작업 동시에 가능)               │
│                                                                 │
│  ✅ 검색 기능: 이전에 입력한 명령어를 쉽게 찾을 수 있어요        │
│                                                                 │
│  ✅ 자동완성: 더 똑똑한 자동완성으로 타이핑이 줄어들어요         │
│                                                                 │
│  ✅ 테마/색상: 눈이 편한 색상 테마를 선택할 수 있어요            │
│                                                                 │
│  ✅ 단축키: 더 많은 단축키로 빠르게 작업할 수 있어요             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**설치 방법:**
1. [iterm2.com](https://iterm2.com)에서 다운로드
2. 또는 Homebrew로: `brew install --cask iterm2`

### 터미널 생태계 알아보기

터미널도 자동차처럼 여러 종류가 있습니다. 비유로 이해해볼까요?

```
┌─────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  📦 기본 터미널 = 자동차 교습용 차                               │
│  └─ Mac Terminal, Windows PowerShell                           │

│                                                                 │
│  🚀 향상된 터미널 = 옵션 많은 신차                               │

│     "네비게이션, 후방카메라, 열선시트까지! 더 편하고 빨라요"       │
│                                                                 │

│  └─ Warp, Amazon Q (구 Fig)                                    │

```

#### 추천 무료 터미널 5선

모두 무료이며, 각각 다른 장점이 있습니다.

| 터미널 | 플랫폼 | 특징 | 추천 대상 |
|--------|--------|------|-----------|
| **iTerm2** | Mac | 화면 분할, 검색, 자동완성, 20년간 검증된 안정성 | Mac 사용자라면 거의 필수 |
| **Windows Terminal** | Windows | MS 공식 앱, 탭 지원, PowerShell/WSL 통합 | Windows 사용자 기본 선택 |
| **Warp** | Mac, Linux | AI 내장, 명령어 자동완성, 현대적 UI, 팀 공유 기능 | AI 도움을 원하는 분 |
| **Alacritty** | 모든 플랫폼 | GPU 가속으로 매우 빠름, 설정 파일로 커스터마이징 | 속도를 중시하는 분 |
| **Hyper** | 모든 플랫폼 | 웹 기술 기반, 플러그인으로 무한 확장, 예쁜 테마 | 커스터마이징을 좋아하는 분 |

```
┌─────────────────────────────────────────────────────────────────┐
│                    터미널 선택 가이드                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  "난 Mac 쓰는데 뭘 써야 해?"                                     │
│  └─→ iTerm2 (가장 무난하고 안정적)                              │
│                                                                 │
│  "Windows인데?"                                                 │
│  └─→ Windows Terminal (MS 스토어에서 무료 설치)                 │
│                                                                 │
│  "AI 도움 받고 싶어"                                            │
│  └─→ Warp (명령어 모를 때 물어볼 수 있음)                       │
│                                                                 │
│  "난 빠른 게 좋아"                                              │
│  └─→ Alacritty (체감 속도가 다름)                               │
│                                                                 │
│  "예쁘게 꾸미고 싶어"                                           │
│  └─→ Hyper (테마와 플러그인 풍부)                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**다운로드 링크:**
- iTerm2: [iterm2.com](https://iterm2.com)
- Windows Terminal: Microsoft Store에서 "Windows Terminal" 검색
- Warp: [warp.dev](https://www.warp.dev)
- Alacritty: [github.com/alacritty/alacritty](https://github.com/alacritty/alacritty)
- Hyper: [hyper.is](https://hyper.is)

> 🔥 **프로 팁**
>

#### AI 코드 에디터: Cursor와 친구들

```
┌─────────────────────────────────────────────────────────────────┐
│        🏠 비유: 집 수리할 때 어떤 도구를 쓸까?                     │

│                                                                 │
│  📞 Claude Code = 전화로 부르는 전문 수리기사                     │

│  → 단, 기사님 오실 때까지 기다려야 함 (터미널 ↔ 에디터 왔다갔다)  │
│                                                                 │
│  🏪 Cursor = 집에 상주하는 AI 집사                               │
│  ──────────────────────────────────────────                     │
│  "이거 고쳐줘" → 바로 옆에서 즉시 고침                            │
│  → 내가 뭔가 하려고 하면 "이렇게 하시려는 거죠?" 미리 제안        │
│  → 모든 게 한 공간에서 일어남 (에디터 안에서 다 해결)             │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  둘 다 같은 일을 해요. 방식이 다를 뿐!                           │
│  - Claude Code: 터미널에서 AI와 대화 → 파일 수정됨               │
│  - Cursor: 에디터에서 AI와 대화 → 파일 수정됨                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 요금제 안내

Claude Code는 유료 서비스입니다.

### API 키 vs 구독: 무엇이 다른가요?

이 두 가지 개념은 처음에 헷갈릴 수 있습니다. 쉽게 설명해 드리겠습니다.

| 구분 | Claude Pro/Max 구독 | API 키 사용 |
|------|---------------------|-------------|
| **비유** | 뷔페 식당 월정액 | 먹은 만큼 계산 |
| **결제 방식** | 월 $20/100 고정 | 사용량에 따라 변동 |
| **사용량 제한** | 있음 (하루/월 한도) | 없음 (잔액까지) |
| **설정 난이도** | 쉬움 (그냥 로그인) | 약간 복잡 (키 등록 필요) |
| **추천 대상** | 일반 사용자, 학습자 | 개발자, 헤비 유저 |

### 구독이 적합한 경우

- Claude Code를 처음 배우는 중
- 하루에 몇 시간 정도만 사용
- 간단한 개인 프로젝트
- 예측 가능한 비용 원함

### API 키가 적합한 경우

- 하루 종일 개발 작업
- 대규모 코드베이스 분석
- 팀 프로젝트에서 공유
- 사용량이 들쭉날쭉

> 💡 **비전공자 팁**
>
> 처음에는 무조건 구독을 추천합니다!
> 1. 설정이 훨씬 간단합니다 (로그인만 하면 끝)
> 2. 비용이 예측 가능합니다
> 3. 나중에 필요하면 API 키로 전환할 수 있습니다

### 사용 방법

1. **Claude Pro/Max 구독**: 월 $20부터. 일반 사용량에 충분합니다.
2. **API 키 사용**: 사용한 만큼 결제. 개발자용.

처음이라면 Claude Pro 구독을 권장합니다. [claude.ai/pricing](https://claude.ai/pricing)에서 확인하세요.

### API 키로 사용하기 (선택)

API 키가 있다면:

**Mac/Linux:**
```bash
export ANTHROPIC_API_KEY="your-api-key"
claude
```

**Windows PowerShell:**
```powershell
$env:ANTHROPIC_API_KEY="your-api-key"
claude
```

> 🔥 **프로 팁**
>
> API 키를 매번 입력하기 귀찮다면, 환경 변수로 영구 설정할 수 있습니다:
> - Mac/Linux: `~/.bashrc` 또는 `~/.zshrc`에 export 문 추가
> - Windows: 시스템 환경 변수에 ANTHROPIC_API_KEY 추가

### API 키 발급 방법

1. [console.anthropic.com](https://console.anthropic.com) 접속
2. 회원가입 또는 로그인
3. 좌측 메뉴에서 "API Keys" 클릭
4. "Create Key" 버튼 클릭
5. 키 이름 입력 (예: "my-claude-code")
6. 생성된 키 복사 (이때만 볼 수 있음!)

> ⚠️ **주의사항**
>
> API 키는 생성 직후 한 번만 표시됩니다.
> 반드시 안전한 곳에 저장하세요!
> 잃어버리면 새로 만들어야 합니다.

Claude는 "파일 목록 보기" 요청을 이해하고, 해당하는 터미널 명령어(`ls` 또는 `dir`)를 실행한 뒤 결과를 보여줍니다.

```
> 지금 어디야?
```

현재 작업 중인 폴더 경로를 알려줍니다.

**테스트 4: 간단한 계산**
```
> 25 곱하기 17은?
```

Claude는 코딩뿐만 아니라 일반적인 질문에도 답합니다.

## 🚨 흔한 에러와 해결법

### 에러 1: "command not found: claude"

**화면에 이렇게 나옴:**
```
zsh: command not found: claude
```
또는
```
'claude' is not recognized as an internal or external command
```

**원인:** 설치 후 터미널을 재시작하지 않음

**해결:**
1. 터미널을 완전히 닫기
2. 새 터미널 열기
3. 다시 `claude --version` 시도

그래도 안 되면:
```bash
# Mac/Linux
source ~/.bashrc
# 또는
source ~/.zshrc
```

### 에러 2: 로그인이 안 됨

**화면에 이렇게 나옴:**
브라우저가 열리지 않거나, 로그인 후에도 터미널이 반응 없음

**해결:**
1. 브라우저에서 [claude.ai](https://claude.ai)에 로그인되어 있는지 확인
2. 다른 브라우저로 시도
3. VPN이 켜져 있으면 끄고 시도

### 에러 3: "Permission denied"

**화면에 이렇게 나옴:**
```
Error: EACCES: permission denied
```

**해결 (Mac/Linux):**
```bash
sudo npm install -g @anthropic-ai/claude-code
```

**해결 (Windows):**
PowerShell을 관리자 권한으로 실행하세요.

### 에러 4: npm 명령어를 찾을 수 없음

**화면에 이렇게 나옴:**
```
npm: command not found
```

**원인:** Node.js가 설치되지 않음

**해결:** 설치 스크립트 방법을 사용하세요 (방법 1)

### 에러 5: 설치 스크립트가 실패함

**화면에 이렇게 나옴:**
```
curl: (7) Failed to connect
```
또는
```
Unable to connect to host
```

**원인:** 인터넷 연결 문제 또는 방화벽

**해결:**
1. 인터넷 연결 확인
2. VPN 끄기
3. 방화벽 일시 비활성화
4. 다른 네트워크에서 시도

### 에러 6: "The term 'claude' is not recognized"

**화면에 이렇게 나옴 (Windows):**
```
The term 'claude' is not recognized as the name of a cmdlet
```

**원인:** 환경 변수 PATH에 Claude가 등록되지 않음

**해결:**
1. PowerShell을 닫고 다시 열기
2. 컴퓨터 재시작
3. 그래도 안 되면 재설치

### 에러 7: API 키 인증 실패

**화면에 이렇게 나옴:**
```
Invalid API key
```
또는
```
Authentication failed
```

**원인:** API 키가 잘못되었거나 만료됨

**해결:**
1. API 키 앞뒤 공백 확인
2. 새 API 키 발급
3. 환경 변수가 제대로 설정되었는지 확인

### 에러 8: 네트워크 타임아웃

**화면에 이렇게 나옴:**
```
Request timeout
```
또는
```
Network error
```

**원인:** 네트워크가 불안정하거나 서버 문제

**해결:**
1. 몇 분 후 다시 시도
2. 다른 네트워크로 변경
3. [status.anthropic.com](https://status.anthropic.com) 확인

### 에러 9: 구독이 만료됨

**화면에 이렇게 나옴:**
```
Subscription expired
```
또는
```
Usage limit exceeded
```

**원인:** Claude Pro 구독이 만료되었거나 한도 초과

**해결:**
1. [claude.ai/settings](https://claude.ai/settings)에서 구독 상태 확인
2. 다음 달까지 기다리거나 플랜 업그레이드
3. API 키 방식으로 전환

### 에러 10: Windows에서 스크립트 실행 정책 오류

**화면에 이렇게 나옴:**
```
execution of scripts is disabled on this system
```

**원인:** Windows 보안 정책이 스크립트 실행을 막음

**해결:**
관리자 권한 PowerShell에서:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 그래도 안 되면

```bash
claude /doctor
```
이 명령어가 문제를 진단합니다.

## ❌ 자주 하는 실수

### 1. 엉뚱한 곳에 입력하기
터미널에 입력하고 있는지 확인하십시오. 텍스트 에디터나 브라우저가 아닙니다. 터미널은 텍스트가 있는 검정색/흰색 창입니다.

> 💡 **비전공자 팁**
>
> 터미널은 보통 검은 배경에 흰 글자, 또는 흰 배경에 검은 글자입니다.
> 명령어를 입력하는 곳에 `$`, `%`, 또는 `>` 기호가 있습니다.

### 2. 명령어 복사 잘못하기
설치 명령어를 복사할 때 전체 줄을 복사했는지 확인하십시오. 가끔 첫 글자나 마지막 글자가 빠질 수 있습니다.

> ⚠️ **주의사항**
>
> 특히 `|` (파이프) 기호가 빠지면 안 됩니다.
> `curl -fsSL https://claude.ai/install.sh | bash` 전체를 복사하세요.

### 3. 설치 완료 안 기다리기
설치에 1~2분 걸릴 수 있습니다. 실행 중에 터미널을 닫지 마십시오. "설치 완료" 같은 메시지가 나올 때까지 기다리십시오.

### 4. 터미널 재시작 안 하기
설치 후에는 터미널을 닫고 다시 열어야 합니다. 그래야 Claude Code가 있다는 것을 인식합니다.

> 🔥 **프로 팁**
>
> Mac에서는 `source ~/.zshrc`로 터미널을 닫지 않고도 새 설정을 적용할 수 있습니다.

### 5. VPN이나 방화벽 문제
로그인이 안 되면 VPN을 끄고 다시 시도해보십시오. 일부 회사 방화벽이 로그인 과정을 막을 수 있습니다.

### 6. 잘못된 터미널 사용
Windows에서 명령 프롬프트(cmd)보다 PowerShell을 권장합니다. Mac에서는 기본 터미널 앱을 사용하세요.

### 7. 여러 터미널 창 혼동
여러 터미널 창을 열어두면 혼란스러울 수 있습니다. 처음에는 하나만 열어두고 연습하세요.

## 🆘 안 되면?

| 증상 | 해결 방법 |
|------|----------|
| 터미널이 안 열림 | Mac: Spotlight에서 "Terminal" 검색 / Windows: 시작 버튼 우클릭 → "PowerShell" |
| 설치 명령어가 실패 | 인터넷 연결 확인, 다른 설치 방법 시도 |
| "claude: command not found" | 터미널을 완전히 닫고 다시 열기 |
| 로그인 페이지가 안 열림 | 먼저 [claude.ai](https://claude.ai)를 직접 열어서 로그인 확인 |
| API 키 오류 | 새 키 발급, 앞뒤 공백 제거 |
| 여전히 막혀있음 | `claude /doctor` 실행, [공식 문서](https://docs.anthropic.com/claude-code) 확인 |

## 💡 현업에서는 이렇게 사용해요

### 실제 개발자들의 Claude Code 활용

1. **코드 리뷰**: "이 코드 리뷰해줘. 문제 있으면 알려줘."
2. **리팩토링**: "이 함수 더 깔끔하게 정리해줘."
3. **문서화**: "이 코드에 주석 달아줘."
4. **테스트 작성**: "이 함수 테스트 코드 만들어줘."
5. **디버깅**: "이 에러 왜 나는지 알려줘."

### 왜 터미널 기반인가요?

현업 개발자들은 이미 터미널을 많이 사용합니다:
- Git 명령어
- npm/yarn 패키지 관리
- 서버 접속 및 관리
- 빌드 및 배포

Claude Code가 터미널에서 동작하면, 기존 워크플로우에 자연스럽게 통합됩니다.
