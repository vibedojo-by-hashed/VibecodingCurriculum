## 왜 필요합니까?

Claude Code를 설치했습니다. 이제 뭘 해야 할까요? 이 챕터에서는 실제로 대화하는 방법과 파일을 안전하게 보호하는 방법을 배웁니다.

**실제 상황:**
- 웹사이트를 만들고 싶은데, Claude가 실수로 중요한 파일을 지우면 안 됨
- 새 프로젝트를 탐색하면서 변경하기 전에 코드를 이해하고 싶음
- 매번 "승인" 클릭 없이 빠르게 수정하고 싶음

### 쉬운 비유: 도우미에게 지시하기

집에 능력 있는 도우미가 있다고 상상해보십시오. 일을 시작하기 전에 몇 가지 규칙을 정하고 싶을 것입니다:

- **"일단 둘러보기만 해요"** = Plan 모드 (읽기 전용)
- **"뭔가 건드리기 전에 저한테 물어봐요"** = Normal 모드 (허락 요청)
- **"청소는 그냥 해도 돼요, 근데 버리기 전에는 물어봐요"** = Accept Edits 모드 (안전한 작업은 자동 승인)

권한 모드는 Claude에게 정해주는 이런 규칙입니다.

## 기본 대화 방법

Claude Code는 채팅처럼 대화합니다. `>` 뒤에 입력하면 됩니다.

### 대화 시작

```bash
# 터미널에서 Claude Code 시작
claude
```

시작하면 다음 화면이 나타납니다:

```
╭────────────────────────────────────────╮
│ Welcome to Claude Code!                │
│                                        │
│ /help for commands, Shift+Tab for mode │
╰────────────────────────────────────────╯

>
```

여기에 입력하면 됩니다.

### 기본 예시

```
> 안녕!
```

```
> 오늘 날씨 어때?
```

```
> 이 폴더에 뭐가 있어?
```

무엇이든 질문할 수 있습니다. 한국어도 잘 이해합니다.

> 💡 **비전공자 팁**
>
> Claude Code는 일상적인 대화도 가능하지만, 코딩 관련 작업에 최적화되어 있습니다.
> "안녕"보다는 "이 프로젝트 설명해줘"처럼 구체적인 요청이 더 유용한 답을 얻을 수 있습니다.

## 권한 모드 (⭐ 필수 이해)

Claude Code는 세 가지 모드가 있습니다. **이것이 가장 중요합니다.**

### 왜 권한 관리가 필요합니까?

LLM은 여러분의 요청을 "해석"해서 실행합니다. 대부분 정확하지만, 가끔 의도와 다르게 해석할 수 있습니다.

예를 들어 "이 폴더 정리해줘"라고 하면:
- 파일을 카테고리별로 분류하는 것일 수도 있고
- 불필요한 파일을 삭제하는 것일 수도 있습니다

그래서 중요한 작업 전에 확인을 받는 것이 안전합니다.

### 권한 모드의 설계 철학

Claude Code의 권한 시스템은 다음 원칙을 따릅니다:

1. **안전 우선**: 기본적으로 모든 변경에 대해 확인을 요청합니다
2. **점진적 신뢰**: 사용자가 신뢰 수준을 조절할 수 있습니다
3. **투명성**: 무엇을 하려는지 항상 보여줍니다
4. **되돌리기 가능**: 실수해도 복구할 수 있습니다

> 🔥 **프로 팁**
>
> 권한 모드는 "Claude를 믿느냐"의 문제가 아닙니다.
> **자신의 요청이 명확한지 확인**하는 안전장치입니다.
> 모호한 요청 → 예상치 못한 결과의 가능성

### 🔵 1. Plan 모드 (가장 안전)

```
📋 Plan 모드
읽기만 가능. 아무것도 변경할 수 없음.
```

**특징:**
- 파일 읽기만 가능
- 파일 생성/수정/삭제 불가
- 터미널 명령어 실행 불가 (읽기 명령은 가능)
- 계획 수립에 최적

**사용 시점:**
- 처음 프로젝트를 살펴볼 때
- 코드 동작 방식을 이해하고 싶을 때
- 변경 전 계획을 세울 때
- 코드 리뷰나 분석만 할 때
- 실수가 두려울 때

**진입 방법:**
```
> /plan
```
또는 `Shift + Tab` 두 번

**실제 대화 예시:**
```
Plan Mode 📋

> 이 프로젝트 구조 설명해줘

이 프로젝트는 React 기반 웹 애플리케이션입니다.

주요 구조:
├── src/
│   ├── components/  # UI 컴포넌트
│   ├── pages/       # 페이지 컴포넌트
│   └── utils/       # 유틸리티 함수
├── public/          # 정적 파일
└── package.json     # 의존성 목록

핵심 파일:
- src/App.js: 메인 앱 컴포넌트
- src/index.js: 엔트리 포인트
```

> 💡 **비전공자 팁**
>
> Plan 모드에서 "파일 만들어줘"라고 해도 만들어지지 않습니다.
> Claude가 "이렇게 만들면 좋겠다"는 계획만 알려줍니다.
> 실제로 만들려면 모드를 바꿔야 합니다.

### 🟡 2. Normal 모드 (기본)

```
🔒 Normal 모드
변경 전에 매번 확인을 요청함.
```

**특징:**
- 파일 읽기/쓰기 가능 (확인 후)
- 터미널 명령어 실행 가능 (확인 후)
- 모든 변경에 대해 승인 요청
- 안전과 효율의 균형

**사용 시점:**
- Claude Code를 처음 배울 때
- 중요한 파일을 다룰 때
- 변경 사항을 하나씩 확인하고 싶을 때
- 프로덕션 환경에서 작업할 때

Claude가 파일을 수정하려고 하면:
```
Claude wants to edit src/app.js
[Allow] [Deny] [Allow all]
```
이렇게 확인을 요청합니다. Allow를 선택하면 수정됩니다.

**승인 옵션 설명:**
| 옵션 | 의미 | 언제 사용? |
|------|------|-----------|
| `y` / Allow | 이번만 허용 | 변경 내용을 확인하고 괜찮을 때 |
| `n` / Deny | 거부 | 원하지 않는 변경일 때 |
| `a` / Allow all | 비슷한 작업 모두 허용 | 반복 작업이 많을 때 |

**실제 대화 예시:**
```
Normal Mode 🔒

> index.html 파일 만들어줘

Creating file: index.html
──────────────────────────────
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <title>내 페이지</title>
</head>
<body>
    <h1>안녕하세요!</h1>
</body>
</html>
──────────────────────────────

Allow this action? [y/n/a]
> y

✓ Created index.html
```

### 🟢 3. Accept Edits 모드 (빠름)

```
⚡ Accept Edits 모드
파일 수정은 자동 승인. 위험한 명령만 확인 요청.
```

**특징:**
- 파일 읽기/쓰기 자동 승인
- 위험한 명령어만 확인 요청 (rm, git push 등)
- 가장 빠른 작업 속도
- 신뢰가 쌓인 후 사용 권장

**사용 시점:**
- 이미 익숙해졌을 때
- 빠르게 작업하고 싶을 때
- Claude를 신뢰할 수 있을 때
- 개인 프로젝트나 실험 환경에서

**자동 승인되는 작업:**
- 파일 생성
- 파일 수정
- 파일 읽기
- 안전한 터미널 명령어 (ls, cat, npm install 등)

**여전히 확인을 요청하는 작업:**
- 파일 삭제 (rm)
- git push
- 시스템 명령어
- sudo 명령어

**실제 대화 예시:**
```
Accept Edits Mode ⚡

> 로그인 폼 컴포넌트 만들어줘

✓ Created src/components/LoginForm.js
✓ Updated src/App.js (import 추가)
✓ Created src/styles/LoginForm.css

로그인 폼 컴포넌트를 생성했습니다. 기능:
- 이메일/비밀번호 입력 필드
- 유효성 검사
- 로그인 버튼

테스트해보시겠어요?
```

> ⚠️ **주의사항**
>
> Accept Edits 모드에서도 실수는 발생할 수 있습니다.
> Git을 사용하면 언제든 이전 상태로 돌아갈 수 있습니다.
> 중요한 프로젝트에서는 Normal 모드를 권장합니다.

### 모드 전환

| 방법 | 설명 |
|------|------|
| `Shift + Tab` | 모드 순환 (Normal → Accept Edits → Plan → Normal) |
| `/plan` | Plan 모드로 전환 |
| `claude --permission-mode plan` | 시작할 때 Plan 모드로 |

### 모드별 비교 표

| 기능 | Plan 📋 | Normal 🔒 | Accept Edits ⚡ |
|------|---------|-----------|-----------------|
| 파일 읽기 | ✅ | ✅ | ✅ |
| 파일 생성/수정 | ❌ | ✅ (확인) | ✅ (자동) |
| 파일 삭제 | ❌ | ✅ (확인) | ✅ (확인) |
| 안전한 명령어 | ✅ | ✅ (확인) | ✅ (자동) |
| 위험한 명령어 | ❌ | ✅ (확인) | ✅ (확인) |
| 작업 속도 | 느림 | 중간 | 빠름 |
| 안전성 | 최고 | 높음 | 중간 |

### 🌟 모드 선택 가이드

```
┌─────────────────────────────────────────────────────┐
│           언제 어떤 모드를 써야 할까요?               │

│                                                     │
│   🆕 처음 프로젝트 열었을 때                         │
│   └─→ Plan 모드로 구조 파악                         │

│   📝 코드 수정이 필요할 때                           │
│   └─→ Normal 모드로 하나씩 확인                     │

│   🚀 빠르게 작업해야 할 때                           │

│   └─→ Plan 모드 유지                               │
│                                                     │
│   🏢 회사 프로젝트                                   │

└─────────────────────────────────────────────────────┘
```

### 🌟 Pro Tip

> **"거의 항상 Plan 모드로 시작하십시오."**

새 작업을 할 때는 `Shift + Tab`을 두 번 눌러서 Plan 모드로 먼저 진입하십시오. 먼저 코드를 분석하고 계획을 세운 뒤 실행하면, "의도와 다르게 해석해서 다시 해야 하는" 상황을 줄일 수 있습니다.

## 자주 사용하는 단축키

외울 필요 없습니다. 필요할 때 확인하면 됩니다.

| 단축키 | 기능 |
|--------|------|
| `Shift + Tab` | 권한 모드 전환 |
| `Ctrl + C` | 현재 작업 취소 |
| `Esc Esc` | 되돌리기 (마지막 상태로) |
| `Ctrl + L` | 화면 정리 (대화는 유지) |
| `Shift + Enter` | 줄바꿈 (긴 메시지 작성 시) |

> 💡 **비전공자 팁**
>
> 가장 중요한 건 `Shift + Tab` (모드 전환)과 `Esc Esc` (되돌리기)입니다.
> 이 두 개만 기억하세요!

### 단축키 상세 설명

#### Shift + Tab - 모드 전환
```
Normal 🔒 → Accept Edits ⚡ → Plan 📋 → Normal 🔒 ...
```
순서대로 순환합니다. 화면 상단에서 현재 모드를 확인할 수 있습니다.

#### Esc Esc - 되돌리기
Claude가 방금 한 작업을 취소합니다.
```
> 파일 내용 바꿔줘
✓ Updated config.js

> (Esc Esc)
↩ Reverted changes to config.js
```

> 🔥 **프로 팁**
>
> `Esc Esc`는 마지막 작업만 되돌립니다.
> 여러 작업을 되돌리려면 Git을 사용하세요.

#### Ctrl + C - 작업 취소
Claude가 응답을 생성하는 중에 멈추고 싶을 때 사용합니다.
```
> 긴 코드 작성해줘

Claude: 네, 작성하겠습니다...
[=========>            ] 50%

(Ctrl + C)

작업이 취소되었습니다.
```

#### Shift + Enter - 줄바꿈
긴 요청을 여러 줄로 나눠서 작성할 수 있습니다.
```
> 웹사이트 만들어줘 (Shift+Enter)
> - 헤더에 로고 넣어줘 (Shift+Enter)
> - 3개 섹션으로 나눠줘 (Shift+Enter)
> - 다크 모드 지원해줘 (Enter로 전송)
```

### Mac 사용자 참고

Mac에서 `Alt` 키 조합을 사용하려면 터미널 설정에서 "Option as Meta"를 활성화해야 합니다.

**설정 방법:**
1. 터미널 > 환경설정 > 프로파일
2. 키보드 탭
3. "Option 키를 메타 키로 사용" 체크

## 슬래시 명령어

`/`로 시작하는 특수 명령어입니다.

| 명령어 | 설명 | 사용 예시 |
|--------|------|----------|
| `/help` | 모든 명령어 보기 | 명령어가 기억 안 날 때 |
| `/clear` | 대화 내용 삭제 | 새로운 주제로 시작할 때 |
| `/exit` | Claude Code 종료 | 작업 끝났을 때 |
| `/cost` | 현재까지 사용 비용 보기 | API 키 사용자 |
| `/model` | 사용 중인 모델 보기/변경 | 다른 모델 사용하고 싶을 때 |
| `/plan` | Plan 모드로 전환 | 읽기 전용 모드 |

### /clear vs Ctrl+L

이 둘은 비슷해 보이지만 완전히 다릅니다!

| 구분 | `/clear` | `Ctrl+L` |
|------|----------|----------|
| 화면 | 정리됨 | 정리됨 |
| 대화 기록 | **삭제됨** | 유지됨 |
| Claude 기억 | **잊어버림** | 기억함 |
| 용도 | 새 주제 시작 | 화면만 깔끔하게 |

**예시:**
```
> 버튼 컴포넌트 만들어줘
✓ Created Button.js

> /clear

> 이전에 만든 버튼 수정해줘
어떤 버튼을 말씀하시는 건가요?
(Claude가 이전 대화를 잊어버림)
```

```
> 버튼 컴포넌트 만들어줘
✓ Created Button.js

> (Ctrl + L)

(화면이 깨끗해짐)

> 이전에 만든 버튼 수정해줘
네, Button.js를 수정하겠습니다.
(Claude가 기억함)
```

> 💡 **비전공자 팁**
>
> 헷갈리면 `Ctrl+L`만 쓰세요.
> `/clear`는 정말로 처음부터 시작하고 싶을 때만 사용합니다.

## 세션 이어서 하기

Claude Code를 종료했다가 다시 시작해도 이전 대화를 이어서 할 수 있습니다.

```bash
# 마지막 대화 이어서
claude --continue

# 과거 대화 중에서 선택
claude --resume
```

**--continue 사용 예시:**

## 좋은 질문 vs 나쁜 질문

### 나쁜 질문 예시

| 질문 | 문제점 |
|------|--------|
| "웹사이트 만들어줘" | 너무 모호함. 어떤 웹사이트? |
| "코드 고쳐줘" | 무엇을 어떻게 고쳐야 하는지 불명확 |
| "안 돼" | 무엇이 안 되는지 설명 없음 |
| "최고로 해줘" | 기준이 불명확 |
| "에러 나" | 어떤 에러인지 정보 없음 |

### 좋은 질문 예시

| 질문 | 왜 좋은가 |
|------|----------|
| "포트폴리오 웹사이트 만들어줘. 소개, 프로젝트 3개, 연락처 섹션으로." | 구체적인 요구사항 |
| "@app.js 35번째 줄 null 에러 고쳐줘" | 파일, 위치, 에러 종류 명시 |
| "'Cannot read property'라는 에러가 나. 콘솔 로그 보여줄게." | 에러 메시지 공유 |
| "모바일에서도 잘 보이게 반응형으로 해줘" | 명확한 기준 |
| "로딩이 3초 이상 걸려. 1초 이내로 줄이고 싶어." | 측정 가능한 목표 |

### 질문 개선 예시

**Before:**
```
> 버그 있어
```

**After:**
```
> @src/api.js에서 사용자 데이터 불러올 때 에러가 나.
> 콘솔에 "TypeError: Cannot read property 'name' of undefined"라고 나와.
> 사용자가 로그인 안 했을 때 발생하는 것 같아.
```

## 심화: 효과적인 Claude Code 활용

### 생각 → 계획 → 실행

AI 도구를 처음 접하면 바로 무언가를 입력하고 싶어집니다. 하지만 잠깐 멈추고 "내가 뭘 만들려고 하지?"를 먼저 정리하면 결과가 훨씬 좋아집니다.

Plan Mode는 이 과정을 돕습니다. Claude가 코드를 분석하고 접근 방법을 제안하면, 여러분은 그 계획이 맞는지 확인한 후 진행할 수 있습니다.

```
┌──────────────────────────────────────────────────┐
│              효과적인 작업 흐름                    │
├──────────────────────────────────────────────────┤
│                                                  │
│  1️⃣ Plan 모드: "이 프로젝트 분석해줘"              │
│     └─→ 구조 파악, 계획 수립                      │
│                                                  │
│  2️⃣ Normal 모드: "이 부분 수정해줘"               │
│     └─→ 중요한 변경은 확인하면서 진행              │
│                                                  │
│  3️⃣ Accept Edits 모드: "나머지 빠르게 해줘"       │
│     └─→ 반복 작업 빠르게 처리                     │
│                                                  │
│  4️⃣ Plan 모드: "지금까지 한 거 검토해줘"          │
│     └─→ 최종 확인                                │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 권한 모드 선택 기준

| 상황 | 권장 모드 | 이유 |
|-----|---------|------|
| 위험 낮음 + 되돌리기 쉬움 | Accept Edits | 속도 우선 |
| 위험 중간 또는 처음 하는 작업 | Normal | 확인하면서 진행 |
| 위험 높음 또는 되돌리기 어려움 | Plan | 먼저 계획 |
| 코드 리뷰 | Plan | 읽기만 필요 |
| 학습 목적 | Normal | 과정 이해 |
| 프로덕션 배포 전 | Plan → Normal | 신중하게 |

## 🚨 흔한 에러와 해결법

### 에러 1: Plan 모드에서 파일이 안 만들어짐

**상황:** "파일 만들어줘"라고 했는데 설명만 하고 안 만듦

**원인:** Plan 모드는 읽기 전용입니다

**해결:** `Shift + Tab`으로 Normal 또는 Accept Edits 모드로 전환

**대화 예시:**
```
Plan Mode 📋

> index.html 만들어줘

Plan 모드에서는 파일을 생성할 수 없습니다.
다음과 같은 구조로 만들면 좋겠습니다:
- DOCTYPE 선언
- head 섹션 (메타, 타이틀)
- body 섹션 (콘텐츠)

실제로 생성하려면 Normal 모드로 전환해주세요 (Shift+Tab).
```

### 에러 2: 실수로 잘못된 파일 수정

**상황:** 원하지 않는 파일이 수정됨

**해결:** `Esc Esc` (Escape 두 번)로 되돌리기. 또는:
```
> 방금 거 취소해줘
```

### 에러 3: 모드 전환이 안 됨

**상황:** `Shift + Tab`을 눌러도 반응 없음

**해결:**
- `Shift`를 누른 상태에서 `Tab`을 누르고 있는지 확인
- 일부 터미널은 키 설정이 다를 수 있음. `/plan` 명령어로 직접 전환

### 에러 4: Claude가 이전 대화를 기억 못함

**상황:** "아까 만든 파일"이라고 했는데 모른다고 함

**원인:** `/clear`를 사용했거나 새 세션을 시작함

**해결:**
```bash
# 이전 대화 이어서
claude --continue
```

### 에러 5: 승인 대화상자가 너무 많이 뜸

**상황:** 파일 하나 만드는데 승인을 여러 번 해야 함

**해결:**
- `a` (Allow all)를 선택하면 비슷한 작업은 자동 승인
- Accept Edits 모드로 전환

## ❌ 자주 하는 실수

### 1. 어떤 모드인지 잊어버리기
요청하기 전에 항상 모드 표시를 확인하십시오.

> 💡 **비전공자 팁**
>
> 화면 상단에 항상 현재 모드가 표시됩니다.
> Plan 📋 / Normal 🔒 / Accept Edits ⚡

### 2. 실수로 "Deny" 클릭하기
실수로 거절했으면, 다시 요청하면 됩니다. "다시 해줘"라고 하면 됩니다.

### 3. 탐색할 때 Plan 모드 안 쓰기
프로젝트를 처음 열 때, 바로 수정하고 싶은 충동을 참으십시오. Plan 모드로 먼저 구조를 파악하십시오.

### 4. 자연어 대신 명령어 입력하기
터미널 명령어를 입력할 필요 없습니다. `ls -la` 대신 "이 폴더에 있는 파일 보여줘"라고 하십시오.

### 5. Claude의 설명 안 읽기
Claude는 무엇을 할 것인지, 왜 하는지 자주 설명합니다. 이 설명을 읽으면 배우기도 하고 문제도 미리 발견할 수 있습니다.

### 6. 모호한 요청하기
"고쳐줘"보다 "null 에러 고쳐줘"가 낫습니다. 구체적일수록 좋습니다.

### 7. 한 번에 너무 많이 요청하기
"쇼핑몰 만들어줘"보다 단계별로 요청하는 것이 좋습니다.

| 모드가 안 바뀜 | `Shift + Tab` 확인, 또는 `/plan` 명령어 사용 |
| Claude가 응답 안 함 | 인터넷 연결 확인, `/help` 입력, `/exit` 후 재시작 |
| 권한 대화상자가 안 나옴 | Plan 또는 Accept Edits 모드일 수 있음. `Shift + Tab`으로 Normal 모드로 전환 |
| 원하지 않는 작업을 함 | `Esc Esc`로 되돌리기 |
| 이전 대화 기억 못함 | `claude --continue`로 이전 세션 복원 |

- [ ] 세 가지 권한 모드를 설명할 수 있다 (Plan, Normal, Accept Edits)
