## 왜 필요합니까?

코딩에서 파일은 전부입니다. 웹사이트, 앱, 도구 등 만드는 모든 것이 파일로 이루어져 있습니다. 파일 다루는 법을 배우는 건 요리에서 재료 다루는 법을 배우는 것과 같습니다--재료 없이는 아무것도 만들 수 없습니다.

**실제 상황:**
- 웹사이트를 만들고 싶다 (HTML, CSS 파일 필요)
- 기존 문서를 수정하고 싶다
- 어지러운 프로젝트 파일을 정리하고 싶다
- Claude가 프로젝트 구조를 이해하게 하고 싶다

### 쉬운 비유: 서류 캐비닛

컴퓨터를 거대한 서류 캐비닛이라고 생각해보십시오. 각 서랍이 폴더이고, 안에 있는 각 종이가 파일입니다.

`@report.txt`라고 말하면, 특정 종이를 가리키면서 "이거 봐"라고 하는 겁니다. `@documents/`라고 말하면, 서랍 전체를 가리키면서 "여기 있는 거 다 확인해"라고 하는 겁니다.

`@` 기호는 Claude가 볼 것을 가리키는 여러분의 손가락입니다.

## 파일 시스템 기초

파일을 다루기 전에 기본 개념을 알아봅시다.

### 파일과 폴더

```
┌──────────────────────────────────────────┐
│           파일 시스템 구조                 │
├──────────────────────────────────────────┤
│                                          │
│  📁 my-project/                          │
│  ├── 📄 index.html     ← 파일            │
│  ├── 📄 style.css      ← 파일            │
│  ├── 📁 src/           ← 폴더 (서브폴더)   │
│  │   ├── 📄 app.js                       │
│  │   └── 📄 utils.js                     │
│  └── 📁 images/        ← 폴더            │
│      ├── 📄 logo.png                     │
│      └── 📄 banner.jpg                   │
│                                          │
└──────────────────────────────────────────┘
```

### 경로 (Path)

파일의 위치를 나타내는 주소입니다.

| 경로 유형 | 예시 | 설명 |
|----------|------|------|
| **절대 경로** | `/Users/홍길동/my-project/index.html` | 최상위부터의 전체 경로 |
| **상대 경로** | `src/app.js` | 현재 위치에서의 경로 |
| **상위 폴더** | `../other-folder/file.txt` | `..`은 상위 폴더를 의미 |

> 💡 **비전공자 팁**
>
> 경로는 "집 주소"와 같습니다.
> - 절대 경로: "서울시 강남구 테헤란로 123" (전체 주소)
> - 상대 경로: "옆 건물 3층" (현재 위치 기준)

### 파일 확장자

파일 이름 뒤에 붙는 `.xxx`는 파일 종류를 나타냅니다.

| 확장자 | 용도 | 예시 |
|--------|------|------|
| `.html` | 웹페이지 구조 | `index.html` |
| `.css` | 웹페이지 스타일 | `style.css` |
| `.js` | 자바스크립트 코드 | `app.js` |
| `.json` | 데이터/설정 | `package.json` |
| `.md` | 마크다운 문서 | `README.md` |
| `.txt` | 일반 텍스트 | `notes.txt` |
| `.py` | 파이썬 코드 | `main.py` |

> 🔥 **프로 팁**
>
> 확장자를 보면 파일의 역할을 짐작할 수 있습니다.
> Claude에게 "이 프로젝트 설명해줘"라고 하면 확장자를 보고 어떤 기술을 쓰는지 파악합니다.

## @로 파일 참조하기

Claude Code에서 `@`를 사용하면 파일을 참조할 수 있습니다.

### 기본 사용법

```
> @index.html 이 파일 설명해줘
```

```
> @src/app.js 이 코드에서 버그 찾아줘
```

```
> @package.json 여기서 사용하는 라이브러리 목록 알려줘
```

### 자동완성

`@`를 입력하고 파일 이름을 입력하면 자동으로 목록이 나타납니다. 화살표로 선택하고 Enter를 누릅니다.

```
> @in (자동완성 목록 표시)
  ├── index.html
  ├── index.js
  └── init.py
```

> 💡 **비전공자 팁**
>
> `@` 치고 아무 글자나 입력해보세요. 자동완성 목록이 뜨면 거기서 고르면 됩니다.
> 파일 이름 외울 필요 없어요!

### 폴더 참조

```
> @src/ 이 폴더 구조 설명해줘
```

```
> @components/ 여기 있는 파일들 뭐하는 거야?
```

폴더 경로 끝에 `/`를 붙이면 폴더 전체를 참조합니다.

### 다양한 @ 사용 예시

**파일 분석:**
```
> @README.md 이 프로젝트가 뭐 하는 건지 설명해줘
```

**코드 리뷰:**
```
> @src/auth.js 보안 문제 있는지 검토해줘
```

**에러 디버깅:**
```
> @app.js 23번째 줄에서 에러 나는데 고쳐줘
```

**비교 분석:**
```
> @old-version.js와 @new-version.js 뭐가 달라졌어?
```

**설정 확인:**
```
> @tsconfig.json 이 설정 맞는지 봐줘
```

## 파일 생성

Claude에게 파일을 생성해달라고 요청합니다.

### 기본 예시

```
> hello.txt 파일 만들어줘. 안에 "안녕하세요"라고 써줘.
```

```
> index.html 파일 만들어줘. 간단한 웹페이지로.
```

### 승인 절차

Normal 모드에서는 Claude가 파일을 생성하기 전에 확인을 요청합니다:

```
Claude wants to create hello.txt
───────────────────────────────────────
안녕하세요
───────────────────────────────────────
[Allow] [Deny]
```

`Allow`를 선택하면 파일이 생성됩니다.

### 폴더와 함께 생성

```
> src/components/Button.js 파일 만들어줘. React 버튼 컴포넌트로.
```

폴더가 없으면 Claude가 폴더도 함께 생성합니다.

**예상 대화:**
```
> src/components/Button.jsx 만들어줘

폴더 'src/components'가 없습니다. 생성할까요?

Creating folder: src/components/
Creating file: src/components/Button.jsx
───────────────────────────────────────
import React from 'react';

export default function Button({ children, onClick }) {
  return (
    <button onClick={onClick}>
      {children}
    </button>
  );
}
───────────────────────────────────────

Allow this action? [y/n/a]
```

### 여러 파일 한 번에 생성

```
> my-website 폴더 만들고 그 안에 index.html, style.css, script.js 만들어줘.
> "내 사이트에 오신 것을 환영합니다"라고 나오는 간단한 페이지로.
```

**예상 결과:**
```
✓ Created folder: my-website/
✓ Created file: my-website/index.html
✓ Created file: my-website/style.css
✓ Created file: my-website/script.js

3개 파일을 생성했습니다.
브라우저에서 index.html을 열어 확인해보세요.
```

> 💡 **비전공자 팁**
>
> 복잡한 구조도 한 번에 요청할 수 있습니다:
> "React 프로젝트 기본 구조 만들어줘"
> Claude가 필요한 폴더와 파일을 알아서 생성합니다.

## 파일 수정

기존 파일을 수정할 수 있습니다.

### 기본 예시

```
> @index.html에서 제목을 "내 웹사이트"로 바꿔줘
```

```
> @app.js에 console.log 추가해줘
```

### 수정 내용 확인 (Diff)

Claude가 수정하기 전에 변경 내용을 보여줍니다:

```diff
- <h1>Hello World</h1>
+ <h1>내 웹사이트</h1>
```

빨간색(-)은 삭제되는 부분, 초록색(+)은 추가되는 부분입니다.

### Diff 읽는 법

```
Editing: src/config.js

────────────────────────────────────────
 1 | const config = {
-2 |   port: 3000,
+2 |   port: 8080,
 3 |   host: 'localhost',
+4 |   debug: true,
 5 | };
────────────────────────────────────────

Allow this edit? [y/n/a]
```

| 기호 | 의미 | 색상 |
|------|------|------|
| `-` | 삭제되는 줄 | 빨간색 |
| `+` | 추가되는 줄 | 초록색 |
| (없음) | 변경 없는 줄 | 흰색/회색 |

> 🔥 **프로 팁**
>
> Diff를 읽는 습관을 들이세요. Claude가 어떤 변경을 하려는지 이해하면:
> 1. 실수를 미리 발견할 수 있습니다
> 2. 점점 코드를 이해하게 됩니다
> 3. 나중에 직접 수정할 수 있게 됩니다

### 부분 수정 vs 전체 재작성

**부분 수정 (권장):**
```
> @config.js에서 port를 8080으로 바꿔줘
```

**전체 재작성:**
```
> @config.js 파일 전체 새로 작성해줘
```

> ⚠️ **주의사항**
>
> "전체 새로 작성"은 기존 내용을 모두 덮어씁니다.
> 부분 수정이 더 안전하고 추적하기 쉽습니다.

## 특수 접두사

`@` 외에도 유용한 접두사들이 있습니다.

| 접두사 | 기능 | 예시 |
|--------|------|------|
| `@` | 파일/폴더 참조 | `@src/app.js` |
| `!` | 명령어 실행 후 결과 표시 | `!ls` |
| `#` | Claude 메모리에 저장 | `# 항상 한국어로 대답해` |

### @ - 파일 참조

이미 배운 내용입니다.

```
> @package.json 의존성 목록 알려줘
```

### ! - 명령어 실행

```
> !ls
```
현재 폴더의 파일 목록을 표시하고, Claude가 그 내용을 인식합니다.

```
> !npm run build
```
빌드 명령을 실행하고 결과를 Claude에게 보여줍니다.

```
> !git status
```
Git 상태를 확인하고 Claude가 분석합니다.

**일반 요청과의 차이:**
| 방식 | 예시 | 차이점 |
|------|------|--------|
| 일반 | "파일 목록 보여줘" | Claude가 명령어 결정, 승인 필요 |
| `!` 접두사 | `!ls -la` | 지정한 명령어 바로 실행, 결과를 Claude가 인식 |

> 💡 **비전공자 팁**
>
> `!`는 명령어를 알 때 사용합니다.
> 모르면 그냥 "파일 목록 보여줘"라고 말하면 됩니다.
> Claude가 알아서 적절한 명령어를 실행합니다.

### # - 메모리에 저장

```
> # 이 프로젝트는 TypeScript를 사용해
```
Claude가 이 정보를 기억하고 이후 대화에서 활용합니다.

```
> # 한국어로 대답해줘
```
이후 모든 응답을 한국어로 합니다.

```
> # 코드에 주석을 상세히 달아줘
```
앞으로 생성하는 코드에 상세한 주석을 추가합니다.

**# 활용 예시:**
```
> # 이 프로젝트 규칙:
> # - 파일명은 케밥케이스 (my-component.js)
> # - 함수명은 카멜케이스 (myFunction)
> # - 컴포넌트명은 파스칼케이스 (MyComponent)
> # - 주석은 한국어로

(이후 Claude가 이 규칙을 따름)
```

> 🔥 **프로 팁**
>
> `#`으로 프로젝트 컨벤션을 알려주면 일관성 있는 코드를 생성합니다.
> 팀 코딩 스타일이 있다면 시작할 때 `#`으로 알려주세요.

```
> @app.js 너무 길어. 여러 파일로 분리해줘.
```

### 파일 분리 (코드 스플리팅)


## 실제 프로젝트 파일 관리

### 프로젝트 시작할 때

**1. 구조 파악:**

## 🎯 미니 퀴즈

### Q1. @는 무엇을 하는 기호인가요?
<details>
<summary>정답 보기</summary>

파일이나 폴더를 참조할 때 사용합니다.
예: `@index.html`은 index.html 파일을 가리킵니다.
</details>

### Q2. !와 @의 차이는 무엇인가요?
<details>
<summary>정답 보기</summary>

- `@`: 파일/폴더 참조 (예: `@src/app.js`)
- `!`: 터미널 명령어 실행 (예: `!npm install`)
</details>

### Q3. #은 어떤 용도로 사용하나요?
<details>
<summary>정답 보기</summary>

Claude의 메모리에 정보를 저장합니다.
예: `# 한국어로 대답해줘` → 이후 계속 한국어로 응답
</details>

### Q4. Diff에서 - 와 +는 무엇을 의미하나요?
<details>
<summary>정답 보기</summary>

- `-` (빨간색): 삭제되는 줄
- `+` (초록색): 추가되는 줄
</details>

### Q5. 폴더를 참조할 때는 어떻게 쓰나요?
<details>
<summary>정답 보기</summary>

폴더 이름 끝에 `/`를 붙입니다.
예: `@src/` → src 폴더 전체 참조
</details>

## 🚨 흔한 에러와 해결법

### 에러 1: "파일을 찾을 수 없음"

**상황:** `@config.json 보여줘` 했는데 없다고 함

**원인:** 파일이 다른 폴더에 있거나, 이름이 다름

**해결:**
- 자동완성 사용 (`@` 입력 후 타이핑)
- 경로 포함: `@src/config.json`
- "config 파일 어디 있어?" 물어보기

**대화 예시:**
```
> @config.json 보여줘

파일을 찾을 수 없습니다: config.json

> config 파일 어디 있어?

config 관련 파일들을 찾았습니다:
- src/config/app-config.json
- src/config/db-config.json

어떤 파일을 보시겠어요?
```

### 에러 2: 파일이 엉뚱한 곳에 생김

**상황:** 파일을 만들었는데 예상과 다른 위치에 있음

**해결:**
- 전체 경로 지정: `my-folder/file.txt 만들어줘`
- 현재 위치 확인: `지금 어디야?`

```
> 지금 어디서 작업 중이야?

현재 작업 폴더: /Users/홍길동/projects/my-app

> src 폴더 안에 utils.js 만들어줘

(src/utils.js가 정확한 위치에 생성됨)
```

### 에러 3: 변경 사항이 안 보임

**상황:** 파일 수정했는데 내용이 안 바뀜

**원인:** 승인을 안 함 (Deny 선택했거나 무시함)

**해결:** `Allow` 또는 `y`를 선택했는지 확인

### 에러 4: 자동완성이 안 됨

**상황:** `@` 입력해도 파일 목록이 안 뜸

**해결:**
- 잠시 기다리기 (로딩 중일 수 있음)
- 한 글자 더 입력하기
- 프로젝트 폴더에서 Claude Code 시작했는지 확인

### 에러 5: 파일이 너무 커서 표시 안 됨

**상황:** 큰 파일 내용 요청하니 일부만 보여줌

**해결:**
```
> @large-file.js 처음 100줄만 보여줘
> @large-file.js 함수 목록만 보여줘
> @large-file.js 요약해줘
```

### 에러 6: 잘못된 파일 수정됨

**상황:** 다른 파일이 수정됨

**해결:**
```
> 방금 거 취소해줘
```
또는 `Esc Esc`로 되돌리기

## ❌ 자주 하는 실수

### 1. 파일 이름 오타
```
> @indx.html 보여줘  (index가 아니라 indx)
```
→ 자동완성을 사용하면 오타를 줄일 수 있습니다.

### 2. 경로 생략
```
> @app.js 보여줘  (실제로는 src/app.js)
```
→ 폴더 경로를 포함해보세요.

### 3. 승인 안 하기
Normal 모드에서 파일 생성/수정 시 반드시 승인해야 합니다.

> 💡 **비전공자 팁**
>
> 승인 요청이 귀찮으면 Accept Edits 모드로 전환하세요.
> 단, 중요한 작업에서는 Normal 모드를 권장합니다.

### 4. 잘못된 파일 만들기
실수로 만든 파일은:
```
> 방금 만든 파일 삭제해줘
```
또는 파일 탐색기에서 직접 삭제하세요.

### 5. 백업 없이 수정하기
중요한 파일 수정 전:
```
> @config.js 백업해줘
```
또는 Git 사용을 권장합니다.

### 6. 큰 파일 전체 요청하기
```
> @10000줄-파일.js 전체 보여줘  (시간 오래 걸림)
```
→ 필요한 부분만 요청하세요.

### 7. 확장자 누락
```
> index 만들어줘  (어떤 종류?)
> index.html 만들어줘  (명확함)
```

## 🆘 안 되면?

| 증상 | 해결 방법 |
|------|----------|
| 파일을 찾을 수 없음 | 자동완성 사용, 경로 포함 |
| 파일이 엉뚱한 곳에 생김 | 전체 경로 지정 |

| 수정이 잘못됨 | `Esc Esc` 또는 "취소해줘" |
