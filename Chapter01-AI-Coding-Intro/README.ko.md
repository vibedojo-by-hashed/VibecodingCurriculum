## 들어가기 전에: 당신을 위한 메시지

**"나는 개발자가 아닌데, 이걸 배울 수 있을까?"**

이 질문을 하고 있다면, 당신은 정확히 이 커리큘럼의 타겟입니다. 이 커리큘럼은 코딩을 전혀 모르는 사람을 위해 만들어졌습니다.

**"코딩은 어렵다"**라는 말을 들어봤을 겁니다. 맞습니다, **예전에는** 어려웠습니다. 하지만 지금은 AI가 있습니다.

### 예전과 지금의 차이

| 예전 (2022년 이전) | 지금 (AI 시대) |
|-------------------|---------------|
| 프로그래밍 언어 문법을 암기해야 함 | AI에게 원하는 것을 말하면 됨 |
| 에러가 나면 직접 검색하고 해결해야 함 | AI에게 "왜 에러나?"라고 물어보면 됨 |
| 코드 한 줄 쓰려면 여러 개념을 알아야 함 | 개념 몰라도 결과물을 만들 수 있음 |
| 배우는 데 몇 년 걸림 | 몇 시간 만에 첫 프로젝트 완성 가능 |

**핵심**: 이제 코딩의 진입 장벽이 거의 사라졌습니다. 이 커리큘럼을 따라가면, 오늘 당장 무언가를 만들 수 있습니다.

## 왜 필요합니까?

AI 코딩을 배우면 해결할 수 있는 실제 상황들을 살펴보겠습니다:

### 상황 1: 사이드 프로젝트

> "개인 포트폴리오 웹사이트가 필요해요. 개발자를 고용하자니 비용이 부담되고, 직접 만들자니 코딩을 몰라요."

**AI 코딩 해결책**: Claude에게 "나를 소개하는 웹사이트 만들어줘. 이름, 경력, 연락처가 있고, 심플한 디자인으로"라고 말하면 됩니다. 30분 안에 완성됩니다.

### 상황 2: 업무 자동화

> "매주 엑셀 파일 100개에서 특정 데이터를 추출해서 정리하는 작업을 해요. 너무 반복적이에요."

**AI 코딩 해결책**: Claude에게 "폴더에 있는 엑셀 파일들을 읽어서, A열과 C열 데이터만 추출해서 하나의 파일로 합쳐줘"라고 말하면, 자동화 스크립트가 만들어집니다.

### 상황 3: 앱 아이디어

> "간단한 할 일 관리 앱 아이디어가 있어요. 기존 앱들은 너무 복잡하고, 내가 원하는 기능만 있는 앱이 필요해요."

**AI 코딩 해결책**: Claude에게 단계별로 요청하면 됩니다. "할 일 추가하는 기능 만들어줘" → "완료 체크 기능 추가해줘" → "삭제 기능도 추가해줘"

### 상황 4: 학습 목적

> "코딩을 배우고 싶은데, 책이나 강의를 봐도 너무 어려워서 포기했어요."

**AI 코딩 해결책**: 일단 만들면서 배웁니다. Claude에게 "이 코드가 뭐 하는 건지 설명해줘"라고 물어보면, 친절하게 설명해줍니다. 이론보다 실습이 먼저입니다.

## 쉬운 비유: 통역사

AI 코딩을 이해하는 가장 쉬운 방법은 **통역사**에 비유하는 것입니다.

### 시나리오: 프랑스어 편지

프랑스에 있는 친구에게 편지를 쓰고 싶습니다. 하지만 프랑스어를 모릅니다.

**예전 방식** (전통적인 코딩 학습):
1. 프랑스어 학원에 등록합니다
2. 6개월~1년간 문법, 어휘, 작문을 배웁니다
3. 실수하면서 고치고, 또 배웁니다
4. 드디어 편지를 씁니다

**AI 방식** (AI 코딩):
1. 프랑스어 잘하는 친구(AI)에게 갑니다
2. "이런 내용의 편지를 프랑스어로 써줘"라고 합니다
3. 친구가 써줍니다
4. "이 부분 좀 더 친근하게 바꿔줘"라고 하면 수정해줍니다

**핵심 차이**:
- 예전: 모든 것을 내가 직접 해야 함
- 지금: AI라는 전문가가 대신해줌

### 그럼 나는 뭘 해야 하나요?

좋은 질문입니다. **여러분이 해야 할 일은 "무엇을 원하는지 명확하게 전달하는 것"**입니다.

"편지 써줘"보다 "내 친구 피에르에게, 작년에 파리에서 만났던 추억을 이야기하고, 올해 한국에 놀러오라고 초대하는 편지 써줘"가 훨씬 좋은 결과를 만듭니다.

이것이 바로 **프롬프팅(Prompting)**입니다. 이 커리큘럼에서 배우게 될 핵심 기술 중 하나입니다.

## AI 코딩이란?

### 정의

**AI 코딩**은 인공지능과 대화하면서 소프트웨어를 만드는 방법입니다.

예전에는 이랬습니다:
```
개발자의 머릿속 아이디어 → 개발자가 직접 코드 작성 → 소프트웨어
```

지금은 이렇습니다:
```
당신의 아이디어 → AI에게 설명 → AI가 코드 작성 → 소프트웨어
```

### 왜 이것이 가능해졌을까요?

Claude 같은 **LLM(Large Language Model, 대규모 언어 모델)**은 인터넷에 있는 수십억 줄의 코드를 학습했습니다.

학습한 곳들:
- **GitHub**: 수천만 개의 오픈소스 프로젝트
- **Stack Overflow**: 개발자들의 질문과 답변 수천만 건
- **기술 블로그와 문서**: 모든 프로그래밍 언어의 공식 문서, 튜토리얼

이 모든 것을 학습했기 때문에, AI는:
- 어떤 언어로든 코드를 작성할 수 있고
- 다양한 스타일과 패턴을 알고 있고
- 에러를 보면 원인과 해결책을 추론할 수 있고
- 복잡한 로직도 구현할 수 있습니다

> 💡 **비전공자 팁**
>
> **LLM(대규모 언어 모델)**이 뭔지 정확히 몰라도 괜찮습니다.
>
> 쉽게 말하면, "인터넷에 있는 거의 모든 글을 읽고 학습한 AI"입니다.
> 코딩뿐만 아니라 글쓰기, 번역, 요약 등 다양한 것을 할 수 있습니다.
>
> 자세한 용어 설명은 [용어 사전](../GLOSSARY.ko.md)을 참고하세요.

### 마법이 아니라 패턴 매칭입니다

AI가 코드를 "이해"하는 것은 아닙니다. **패턴을 인식하는 것**입니다.

**예시**: "파란색 제목이 있는 웹페이지 만들어줘"라고 말하면

AI는 이렇게 생각합니다:
1. "파란색 제목" → 이전에 본 수많은 코드 중에서 `color: blue` 또는 `text-blue-500` 패턴과 연결
2. "웹페이지" → HTML 구조 패턴 (`<html>`, `<head>`, `<body>`)
3. 이 패턴들을 조합해서 코드를 생성

결과:
```html
<!DOCTYPE html>
<html>
<head>
    <title>내 페이지</title>
    <style>
        h1 { color: blue; }
    </style>
</head>
<body>
    <h1>안녕하세요</h1>
</body>
</html>
```

**중요**: AI는 패턴을 조합하기 때문에, 요청이 명확할수록 결과도 정확합니다.

## 전통적 코딩 vs AI 코딩: 자세한 비교

### 예시: 간단한 웹페이지 만들기

**목표**: 파란색 제목 "안녕하세요"와 그 아래 "제 이름은 홍길동입니다"가 있는 웹페이지

#### 전통적 방식

먼저 HTML이 무엇인지 배워야 합니다:
- HTML 태그란 무엇인가 (`<html>`, `<head>`, `<body>` 등)
- 태그의 구조와 문법
- 열고 닫는 규칙

그 다음 CSS를 배워야 합니다:
- CSS란 무엇인가
- 선택자(Selector)란 무엇인가
- 속성(Property)과 값(Value)

그리고 직접 코드를 작성합니다:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>내 페이지</title>
    <style>
        body {
            font-family: 'Noto Sans KR', sans-serif;
            margin: 0;
            padding: 20px;
        }
        h1 {
            color: blue;
            font-size: 32px;
        }
        p {
            color: #333;
            font-size: 18px;
        }
    </style>
</head>
<body>
    <h1>안녕하세요</h1>
    <p>제 이름은 홍길동입니다.</p>
</body>
</html>
```

이 코드를 작성하려면:
- HTML 기본 구조 이해: 2-3시간
- CSS 기본 이해: 2-3시간
- 직접 작성하고 테스트: 1시간
- **총 소요 시간: 약 5-7시간** (처음 배우는 경우)

#### AI 코딩 방식

Claude에게 이렇게 말합니다:

```
파란색 제목으로 "안녕하세요"가 나오고,
그 아래 "제 이름은 홍길동입니다"가 나오는 웹페이지 만들어줘
```

Claude가 위와 비슷한 코드를 생성합니다.

- **총 소요 시간: 30초**

#### 차이점 요약

| 항목 | 전통적 방식 | AI 방식 |
|------|-----------|--------|
| 필요한 사전 지식 | HTML, CSS 기본 | 없음 |
| 소요 시간 | 몇 시간 ~ 며칠 | 몇 초 ~ 몇 분 |
| 에러 발생 시 | 직접 검색하고 해결 | "왜 에러나?" 물어보기 |
| 수정 요청 | 직접 코드 수정 | "제목 더 크게 해줘" |
| 학습 곡선 | 가파름 | 완만함 |

## 바이브코딩(Vibecoding)이란?

### 용어의 유래

**Vibe(바이브)**는 "느낌", "분위기"라는 뜻입니다.

**Vibecoding(바이브코딩)**은 "느낌으로 코딩한다"는 의미입니다. 정확한 문법이나 규칙을 몰라도, 원하는 "느낌"을 AI에게 전달하면 AI가 이해하고 구현합니다.

이 용어는 Andrej Karpathy(전 Tesla AI 책임자)가 처음 사용했습니다.

### 바이브코딩의 3단계 사이클

바이브코딩은 다음 3단계를 반복합니다:

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   1. 의도 전달  →  2. 결과 확인  →  3. 피드백으로 수정      │
│        ↑                                           │        │
│        └───────────────────────────────────────────┘        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 1단계: 의도 전달

원하는 것을 자연어로 설명합니다.

```
> 버튼을 클릭하면 숫자가 1씩 올라가는 카운터 만들어줘
```

#### 2단계: 결과 확인

AI가 만든 결과물을 확인합니다. 원하는 대로 됐는지, 아닌지 판단합니다.

```
(Claude가 코드를 생성)
(브라우저에서 확인)
"음, 버튼은 되는데 숫자가 너무 작네..."
```

#### 3단계: 피드백으로 수정

부족한 점을 피드백합니다.

```
> 숫자 크기를 더 크게 해줘. 48px 정도로.
> 그리고 버튼 색깔을 파란색으로 바꿔줘.
```

이 사이클을 몇 번 반복하면, 원하는 결과물이 완성됩니다.

### 바이브코딩의 핵심 원칙

#### 원칙 1: 완벽한 첫 시도는 없다

첫 번째 결과가 완벽할 확률은 낮습니다. 그게 정상입니다. 대화하듯이 수정해나가면 됩니다.

#### 원칙 2: 작게 시작한다

```
# 나쁜 예
> 로그인, 회원가입, 비밀번호 찾기, 프로필 수정 기능이 있는 사용자 관리 시스템 만들어줘

# 좋은 예
> 먼저 로그인 폼만 만들어줘. 이메일이랑 비밀번호 입력하는 것만.
```

작은 단위로 나눠서 하나씩 완성해가는 것이 좋습니다.

#### 원칙 3: 구체적으로 말한다

```
# 모호한 예
> 예쁜 버튼 만들어줘

# 구체적인 예
> 파란색 배경에 흰색 글씨로 "제출"이라고 쓰인 버튼 만들어줘.
> 모서리는 둥글게 하고, 마우스 올리면 좀 더 진한 파란색으로 바뀌게.
```

구체적일수록 원하는 결과에 가깝습니다.

#### 원칙 4: AI는 파트너다

AI는 도구가 아니라 협업 파트너입니다. 질문하고, 의견을 물어보고, 제안을 요청할 수 있습니다.

```
> 이 기능 만들려면 어떤 방법이 좋을까?
> 이 두 가지 방법 중에 뭐가 더 나아?
> 이거 더 좋은 방법 없어?
```

## Claude Code란?

### 정의

**Claude Code**는 Anthropic이 만든 AI 코딩 도구입니다. 터미널(명령어 입력창)에서 실행되며, 대화를 통해 코드를 작성하고, 파일을 생성/수정하고, 명령어를 실행할 수 있습니다.

### 웹 채팅과 Claude Code의 차이

ChatGPT, Claude.ai 같은 웹 채팅에서도 코드를 생성할 수 있습니다. 하지만 중요한 차이가 있습니다.

#### 웹 채팅의 한계

1. **코드를 직접 실행할 수 없음**
   - 웹 채팅: 코드를 보여주기만 함 → 복사해서 파일에 붙여넣어야 함
   - Claude Code: 파일을 직접 생성하고 저장함

2. **현재 상황을 모름**
   - 웹 채팅: 프로젝트 구조를 매번 설명해야 함
   - Claude Code: 폴더와 파일을 직접 읽고 파악함

3. **명령어 실행 불가**
   - 웹 채팅: "npm install 해" → 명령어만 알려줌, 직접 실행해야 함
   - Claude Code: 명령어를 직접 실행하고 결과를 보여줌

4. **연속적인 작업이 어려움**
   - 웹 채팅: 이전 대화 맥락을 기억하지만, 파일 상태는 모름
   - Claude Code: 실제 파일 상태를 보면서 연속 작업 가능

#### 비유로 이해하기

| 웹 채팅 | Claude Code |
|---------|-------------|
| 레시피를 알려주는 요리 유튜버 | 직접 주방에서 요리해주는 셰프 |
| "이렇게 하면 됩니다"라고 말만 함 | 직접 재료 손질하고 요리함 |
| 결과물은 내가 직접 만들어야 함 | 결과물이 바로 나옴 |

### 다른 AI 코딩 도구들과의 비교

Claude Code만 있는 것은 아닙니다. 다른 인기 있는 도구들도 있습니다:

| 도구 | 유형 | 장점 | 단점 |
|------|------|------|------|
| **Claude Code** | CLI (터미널) | 모든 환경에서 작동, 가볍고 빠름, 학습 전이 가능 | 터미널에 익숙해야 함 |
| **Cursor** | IDE (에디터) | VS Code 기반, 시각적인 편집, 코드 자동완성 | 무거움, 특정 IDE에 종속 |
| **Windsurf** | IDE (에디터) | AI 중심 설계, 코드 이해가 깊음 | 새로운 도구라 생태계가 작음 |
| **GitHub Copilot** | 확장기능 | 기존 에디터에서 사용, 코드 자동완성에 특화 | 대화형 작업에는 제한적 |

### 왜 Claude Code를 배우나요?

1. **범용성**: 모든 터미널에서 작동합니다. Mac, Windows, Linux 모두 가능합니다.

2. **가벼움**: 별도의 에디터를 설치할 필요 없이, 기존에 쓰던 에디터와 함께 사용할 수 있습니다.

3. **학습 전이**: CLI 기반 AI 도구의 원리를 이해하면, 다른 도구로 옮겨가도 쉽게 적응할 수 있습니다.

4. **직접 실행**: 파일 생성, 수정, 명령어 실행까지 모두 가능합니다.

> 💡 **비전공자 팁**
>
> **CLI**가 뭔지 모르셔도 됩니다.
>
> 간단히 말하면, 마우스 대신 키보드로 글자를 입력해서 컴퓨터에게 명령하는 방식입니다.
> "터미널"이라고도 부릅니다. Chapter 02에서 자세히 배웁니다.

## 시작 전 마음가짐: 성공을 위한 4가지 원칙

### 원칙 1: 완벽하지 않아도 됩니다

첫 시도에서 완벽한 결과가 나올 확률은 낮습니다. 그게 정상입니다.

**실제 상황**:
```
나: 버튼 만들어줘
Claude: (버튼을 만듦)
나: 음, 색깔이 마음에 안 드네. 파란색으로 바꿔줘.
Claude: (파란색으로 수정)
나: 크기도 좀 더 크게.
Claude: (크기 수정)
```

이렇게 대화하면서 완성해가는 것이 자연스러운 과정입니다.

### 원칙 2: 에러는 당연합니다

코딩에서 에러는 **실패가 아니라 과정**입니다.

전문 개발자도 하루에 수십 번 에러를 마주합니다. 에러가 났다고 좌절할 필요 없습니다.

**에러가 나면**:
```
나: 에러 났어. 이런 메시지가 나와: "TypeError: ..."
Claude: 아, 이 에러는 ... 때문이에요. 이렇게 수정하면 됩니다.
(수정)
```

Claude가 에러도 수정해줍니다.

### 원칙 3: 작게 시작하세요

처음부터 복잡한 것을 만들려고 하지 마세요.

**나쁜 시작**:
```
> 네이버 같은 포털 사이트 만들어줘
```

**좋은 시작**:
```
> "Hello World"가 나오는 웹페이지 만들어줘
```

작은 것부터 시작해서 하나씩 기능을 추가해가세요.

### 원칙 4: 궁금하면 물어보세요

이해가 안 되는 부분이 있으면 그냥 넘어가지 마세요.

```
> 이 코드에서 'const'가 뭐야?
> 왜 여기에 세미콜론을 넣어야 해?
> 이 부분이 이해가 안 돼. 더 쉽게 설명해줘.
```

Claude는 친절하게 설명해줍니다. 질문은 부끄러운 것이 아닙니다.

### 예시 1: 자기소개 웹페이지

```

    그 아래에 "개발자를 꿈꾸는 대학생입니다"라는 소개도 넣어줘.
```

### 예시 3: 코드 디버깅

```
나: 이 코드 실행하면 에러가 나.

const name = "홍길동"
console.log(Name)

Claude: 에러 원인을 찾았어요. JavaScript는 대소문자를 구분합니다.

변수 이름은 'name'(소문자)인데, 출력할 때 'Name'(대문자)로 썼네요.

이렇게 수정하면 됩니다:
console.log(name)  // 소문자 'n'

나: 아, 그렇구나. 대소문자 구분하는 거 처음 알았어.

Claude: 네, JavaScript에서는 'name', 'Name', 'NAME'이 모두 다른 변수예요.
처음에는 헷갈릴 수 있는데, 일반적으로 변수는 소문자로 시작하는
camelCase(카멜케이스) 스타일을 많이 써요.
예: userName, isLoggedIn, totalPrice
```

**핵심**: 코딩 지식이 없어도 대화만으로 해결됩니다.

## ❌ 자주 하는 실수

### 실수 1: "코딩을 먼저 배워야 한다"고 생각하기

**잘못된 생각**: "일단 Python 기초부터 배우고 Claude Code를 써야지"

**올바른 접근**: Claude Code를 먼저 사용하면서 필요한 개념을 배웁니다. 이론보다 실습이 먼저입니다.

### 실수 2: 첫 번째 결과가 완벽할 거라고 기대하기

**잘못된 기대**: "한 번에 원하는 대로 나오겠지?"

**현실**: 첫 결과는 80% 정도입니다. 나머지 20%는 피드백으로 수정합니다.

### 실수 3: 너무 막연하게 요청하기

**나쁜 요청**: "멋진 웹사이트 만들어줘"

**좋은 요청**: "메인 페이지에 큰 제목, 3개의 서비스 소개 카드, 하단에 연락처가 있는 랜딩 페이지 만들어줘. 색상은 파란색 계열로."

### 실수 4: 에러 한 번에 포기하기

**나쁜 반응**: "에러 났네... 난 역시 안 돼..."

**좋은 반응**: "에러 났어. 이런 메시지가 나와: [에러 메시지]. 왜 그런 거야?"

**A**: 정상입니다! 한 챕터씩 천천히 진행하세요. 모든 것을 한 번에 이해할 필요 없습니다. 직접 해보면서 자연스럽게 익혀집니다.

### Q: 제가 이걸 할 수 있을지 불안합니다

**A**: 이메일을 보낼 수 있다면 Claude Code도 사용할 수 있습니다. 대화하듯이 요청하면 됩니다. 특별한 능력이 필요하지 않습니다.

### Q: 무언가를 망가뜨릴까 걱정됩니다

**A**: Claude Code는 안전합니다. 파일을 변경하기 전에 항상 허락을 구합니다. 거절하면 변경되지 않습니다. 실수해도 되돌릴 수 있습니다.

## ✅ 체크리스트

이 챕터를 마치기 전에 다음을 확인하세요:

### 개념 이해
- [ ] AI 코딩이 무엇인지 한 문장으로 설명할 수 있다
- [ ] 바이브코딩의 3단계 사이클(의도 전달 → 결과 확인 → 피드백)을 이해했다
- [ ] 전통적 코딩과 AI 코딩의 차이점을 2개 이상 말할 수 있다

### Claude Code 이해
- [ ] Claude Code가 웹 채팅과 어떻게 다른지 안다
- [ ] Claude Code로 할 수 있는 것을 3가지 이상 알고 있다
- [ ] 다른 AI 코딩 도구들(Cursor, Copilot 등)이 있다는 것을 안다

### 마음가짐
- [ ] 첫 결과가 완벽하지 않아도 괜찮다는 것을 이해했다
- [ ] 에러는 실패가 아니라 과정이라는 것을 이해했다
- [ ] 작게 시작해야 한다는 것을 이해했다
